# Claude Code Agents 完整使用手册

## 概述
Claude Code 提供了 80+ 个专业化的 AI agents，每个都针对特定任务进行了优化。这些 agents 可以自主执行复杂的多步骤任务，大大提升开发效率。

## 核心使用原则

### 1. 自动触发 vs 手动调用
- **自动触发**：当任务匹配 agent 描述时，Claude 会主动使用相应的 agent
- **手动调用**：你可以明确要求使用特定的 agent
- **并发执行**：可以同时启动多个 agents 以提高效率

### 2. Agent 的限制
- 每个 agent 调用是无状态的
- Agent 完成后只返回一次结果
- 需要在初始提示中提供详细的任务描述
- Agent 的输出应该被信任

- 提到"性能"→ 自动使用 performance-engineer

## 最佳实践

### 1. 选择正确的 Agent
- **不确定时**：使用 general-purpose
- **特定语言**：使用对应的语言专家（如 python-pro）
- **特定任务**：使用专业 agent（如 database-optimizer）

### 2. 提供详细的任务描述
```markdown
好的示例：
"用 python-pro agent 重构这个数据处理脚本，
优化内存使用，添加异步处理，
并确保代码符合 PEP 8 规范"

不好的示例：
"优化代码"
```

### 3. 利用 Agent 的专长
每个 agent 都有特定的优势领域，充分利用它们：
- **security-auditor**：不仅审查代码，还能实现安全措施
- **performance-engineer**：不仅找出瓶颈，还能实现优化
- **test-automator**：不仅创建测试，还能设置 CI 管道

### 4. 组合使用
复杂任务可以组合多个 agents：
```markdown
"开发新功能流程：
1. backend-architect 设计 API
2. database-optimizer 设计数据库
3. frontend-developer 创建 UI
4. test-automator 编写测试
5. deployment-engineer 设置部署"
```

## 特殊注意事项

### 1. Context Manager 的使用
- **必须使用场景**：项目超过 10k tokens
- **功能**：协调多个 agents，保持上下文连续性
- **示例**：大型重构、复杂的多步骤任务

### 2. 主动型 Agents
这些 agents 会在特定情况下自动触发：
- **code-reviewer**：写完代码后
- **architect-reviewer**：架构变更后
- **debugger**：遇到错误时
- **incident-responder**：生产问题时

### 3. 状态管理
- 每个 agent 调用是独立的
- 不能向 agent 发送后续消息
- 需要在初始提示中包含所有必要信息

## 故障排除

### 问题：Agent 没有返回预期结果
**解决方案**：
1. 确保任务描述足够详细
2. 检查是否选择了正确的 agent
3. 考虑将任务分解为更小的部分

### 问题：不确定使用哪个 Agent
**解决方案**：
1. 使用 general-purpose agent
2. 让 Claude 自动选择
3. 查看 agent 描述中的使用场景

### 问题：需要多个 Agents 协作
**解决方案**：
1. 使用 context-manager 协调
2. 明确指定执行顺序
3. 利用并发执行提高效率

- 提供清晰的任务描述

- 信任 agents 的输出

随着使用经验的积累，你会越来越熟练地利用这些 agents 来加速你的开发工作流程。
