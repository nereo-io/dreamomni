# BaziAI Next.js 项目架构

## 目录结构

```
项目根目录
├── app/                       # Next.js App Router 页面和API路由
│   ├── [locale]/              # 多语言页面
│   ├── (legal)/               # 法律页面（隔离路由组）
│   └── api/                   # API路由
├── auth/                      # 认证相关
├── components/                # React组件
│   ├── blocks/                # 页面区块组件
│   ├── ui/                    # UI组件 (Shadcn UI)
│   ├── readers/               # 占卜阅读相关组件
│   ├── dashboard/             # 仪表盘组件
│   └── console/               # 控制台组件
├── contexts/                  # React Context
├── hooks/                     # 自定义Hooks
├── i18n/                      # 国际化
│   ├── blocks/                # 区块组件的翻译
│   ├── pages/                 # 页面特定翻译
│   └── messages/              # 全局消息
├── lib/                       # 工具库函数
├── models/                    # 数据模型
├── providers/                 # 提供者组件
├── public/                    # 静态资源
│   └── imgs/                  # 图片资源
├── services/                  # 业务逻辑
│   └── api/                   # API服务
└── types/                     # TypeScript类型定义
    ├── blocks/                # 区块组件类型
    └── pages/                 # 页面类型
```

## 主要技术栈

- **前端框架**: Next.js (App Router)
- **UI框架**: React + Tailwind CSS + Shadcn UI
- **状态管理**: React Context
- **国际化**: next-intl
- **认证**: next-auth
- **数据库**: Supabase
- **支付**: Stripe
- **UI组件**: 
  - 轮播图: Embla Carousel
  - 通知: Sonner
  - 图标: Lucide React
  - Markdown: React Markdown

## 应用结构

### 页面组织
- `app/[locale]/(default)/` - 主要页面
- `app/[locale]/(admin)/` - 管理员页面
- `app/[locale]/auth/` - 认证页面
- `app/[locale]/chat/` - 聊天页面
- `app/[locale]/pay-success/` - 支付成功页面
- `app/(legal)/` - 法律页面

### API路由
- `/api/auth/` - 认证API
- `/api/chat-completion/` - 聊天完成API
- `/api/checkout/` - 结账API
- `/api/membership/` - 会员API
- `/api/readings/` - 占卜阅读API

### 组件分类
- **blocks/**: 页面区块组件，主要用于网站主页和内容页面
- **ui/**: 基础UI组件，来自Shadcn UI
- **readers/**: 占卜阅读相关组件
- **dashboard/**: 用户仪表盘组件
- **console/**: 管理控制台组件

### 国际化
- 支持多种语言: 英文(en)、中文(zh/zh-tw)、日文(ja)、韩文(ko)、法文(fr)
- 使用next-intl实现国际化
- 按区块和页面组织翻译文件

### 数据模型
- 用户(user)
- 会员(membership)
- 占卜阅读(reading)
- 问题(question)
- 订单(order)
- 聊天(chat)

## 业务模块

### 占卜/AI读者
- 使用AI提供占卜服务
- 支持多种占卜类型
- 支持中国生肖和其他占卜形式

### 会员系统
- 使用Stripe处理支付
- 支持会员等级和权限管理
- 管理用户订阅状态

### 国际化
- 支持多语言内容
- 按地区提供定制化服务和内容

## 部署
- 使用Vercel作为主要部署平台
- 支持Cloudflare部署
