# 图片资产规范与生产流程（OG/首图/内文图）

目标：统一图片尺寸、命名、来源、生成与压缩流程，确保页面质量与加载速度，同时满足 SEO（OG、alt、相关性）。

## 目录与命名
- OG 图（社交分享）
  - 目录：`public/og/{section}/{slug}.png`（如 `public/og/compare/veo-3-1-vs-sora-2.png`）
  - 尺寸：1200×630（1.91:1），PNG 或高质量 WebP
  - 命名：与页面 slug 一致；仅小写/连字符
- 首图 / Hero（正文首屏）
  - 目录：`public/images/articles/{type}/{slug}/hero.webp`
  - 尺寸：1600×900（16:9），WebP；同时导出 800×450 作为小屏回退
- 内文插图
  - 目录：`public/images/articles/{type}/{slug}/{index}.webp`（从 `01.webp` 递增）
  - 宽度：优先 1200 px，等比缩放；格式 WebP
- 品牌/Logo
  - 目录：`public/brand/{name}.png`，透明底，较长边 ≤ 512px

## 元信息与无障碍
- 每张图必须提供 `alt` 文案（≤ 120 字），包含对象与场景关键词，但不堆叠
- 首图文件名建议包含核心词，如 `veo-3-1-vs-sora-2-hero.webp`
- 使用 `next/image`，首图可加 `priority`，其它启用懒加载

## 素材来源（合规优先级）
1) 自产素材：
   - 从示例视频截帧：`ffmpeg -ss 00:00:02 -i input.mp4 -frames:v 1 -filter:v scale=1600:-1 frame1.webp`
   - 产品界面/流程截图（注意隐私与品牌授权）
2) 合法开放素材：
   - Unsplash / Pexels / Pixabay（保留作者与许可链接）
3) 生成式图片（可控）：
   - 通过内部模型生成统一风格背景；避免可识别商标与人物肖像

## 统一风格（建议）
- 画幅：首图 16:9；OG 1.91:1；内文图以 16:9/1:1 为主
- 视觉：深色背景 + 浅色标题条/角标；保留 5% 边距
- 水印：右下角 `Veo3 AI`，透明 70%，尺寸随图宽度 6–8%

## 生产流程（团队协作）
1) 选题表绑定 `heroAsset` 与 `assets`
2) 素材收集：截帧/截图/图库/生成式
3) 规范化处理：
   - 批量压缩：`npx squoosh-cli --oxipng auto public/images/.../*.png`
   - WebP 转换：`npx squoosh-cli --webp auto input.png -d output_dir`
   - 批量重命名：与 slug 对齐
4) 质检：
   - 分辨率/格式/体积（首图 ≤ 300 KB，内文图 ≤ 200 KB）
   - alt 文案与无障碍
   - 版权与来源记录（文件侧写入注释或在选题表 sources 字段记录）

## OG 图生成（两种方案，择一）
- 轻量方案（先行）：用 Figma/Canva 模板批量导出 1200×630，填标题与模型名
- 工程方案（后续）：
  - 使用 `@vercel/og` 或 `satori` 动态生成 `/opengraph-image`，从页面数据读取标题/副标题/背景
  - 生成文件落盘 `public/og/...` 作为缓存，或运行时生成（注意配额）

## 视频转图片（从演示视频提帧）
- 取 2–3 张关键帧作为插图或图文对比：
  - `ffmpeg -i input.mp4 -vf "select='eq(n,1)+eq(n,30)+eq(n,60)',scale=1600:-1" -vsync 0 frame_%02d.webp`

## 审核清单（上线前）
- 目录/命名规范、分辨率、体积合格
- alt 文案齐备、与段落相关
- OG 存在且链接正确；首图渲染正常
- 版权许可记录到位；敏感元素已移除/打码
