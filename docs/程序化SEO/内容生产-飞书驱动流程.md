# 内容生产结论（飞书驱动 · GPT 仅产稿 · 本地入库）

本结论固化与你的决定：
- 不产 JSON：GPT 只输出“可读稿件（Markdown）+ 可选的 YouTube 链接/封面图 URL”。
- 不走 Webhook：不再使用 `/api/outrank/webhook`；改为本地脚本直接写数据库。
- 飞书多维表格（Bitable）作为“单一真相源”，脚本周期性读取→清洗→入库 Supabase。

## 生产流水线（5 步）
1) 选题与排期（你+我）
- 在飞书表新增一行：确定 PageType、Title、Locale、素材源等。

2) GPT 生成首稿（仅 Markdown）
- 填充飞书表字段：`ContentMarkdown`（正文仅 Markdown）、`YouTubeURL`（可选）、`CoverImageURL`（可选）。
- 约束：含首屏摘要、主体结构遵循《三类页面大纲模板》，FAQ≥3，附 Sources 与 UpdatedAt。

3) 人审与事实核验（编辑/你）
- 校对参数/价格/功能，补“独特段落/案例”；必要时完善 `MetaDesc/Tags/Slug`。

4) 本地脚本读取飞书→规范化→写库（直连 Supabase）
- 生成缺失字段：`slugify(title)`、`meta_description`（正文首段 150–160 字）、`content_html`（markdown-it 转换）。
- 图片兜底：无 `CoverImageURL` 时尝试用 YouTube 缩略图（转 WebP/压缩）或模板图。
- 幂等写库：以飞书 `record_id` 映射到 `posts.outrank_id`，存在则 `updatePost`，否则 `insertPost`。

5) 发布与观测
- 飞书 `Status=发布` 的记录才会入库为 `Online`；两周复盘收录/展现/CTR。

## 飞书多维表字段（建议）
- Status：拟稿 | 审核 | 发布 | 下线
- PageType：comparison | alternatives | best-of | how-to | review
- Locale：en | zh | …
- Title：文章标题（≤60）
- Slug：可留空（脚本自动生成 kebab-case）
- MetaDesc：可留空（脚本从首段提取 150–160 字）
- Tags：逗号分隔
- CoverImageURL：首图 URL（可空）
- YouTubeURL：可空，脚本校验提取视频 id
- ContentMarkdown：GPT 产稿（仅 Markdown，不含 HTML）
- Sources：多行“标题 | URL | 日期”
- UpdatedAt：YYYY-MM-DD
- RelatedModels / RelatedTools / Category（可选）

备注：飞书 `record_id` 将写入 `posts.outrank_id` 以便幂等更新。

## 数据库映射（Supabase posts）
- title ← Title
- slug ← Slug（空则脚本生成）
- description / meta_description ← MetaDesc（空则脚本生成）
- content / content_markdown ← ContentMarkdown
- content_html ← 脚本通过 markdown-it 渲染得到（可选）
- cover_url / image_url ← CoverImageURL（或兜底图）
- tags ← Tags（string[]）
- locale ← Locale
- status ← Status（发布→online，拟稿/审核→created， 下线→offline）
- outrank_id ← 飞书 record_id

## 本地脚本规范（建议）
- 文件：`scripts/import-lark-posts.ts`
- 能力：
  - 读取飞书表（分页）→ 过滤 `Status=发布`
  - 规范化/校验（slug、描述、YouTube、Sources、最小信息量）
  - 直连 Supabase 写入（insert/update），支持 `--dry-run`、`--update`
- 运行示例：
  - `pnpm ts-node scripts/import-lark-posts.ts --base $LARK_BASE_ID --table $LARK_TABLE_ID --locale en --status 发布 --dry-run`

## 环境变量（.env.local）
- LARK_APP_ID, LARK_APP_SECRET（获取 tenant_access_token）
- LARK_BASE_ID, LARK_TABLE_ID[, LARK_VIEW_ID]
- SUPABASE_URL
- SUPABASE_SERVICE_ROLE_KEY（用于服务器端写入；或本地临时使用）

## 质量门槛与 noindex 规则
- 上线前自检见《发布自检清单》；信息量 <900 字或素材缺失→先不发布或发布为 created（不展示）。

## 后续可选增强
- 动态/批量 OG 图生成并回写飞书；
- 自动从 YouTube 下载缩略图并压缩为 WebP；
- 脚本校验未达标项并在飞书回写错误提示（如缺 FAQ、无来源）。

以上为最终约定的内容生产流程与技术落地方向，可直接执行。
