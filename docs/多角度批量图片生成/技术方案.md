# 多角度批量图片生成技术方案

> **开发状态**: 🔧 Bug 修复中 - 并发回调覆盖问题 (2025-12-05)
>
> **构建验证**: `pnpm build` 通过
>
> **已知问题**: ❌ Agent 模式并发回调时存在数据覆盖 bug（6 张图同时回调会相互覆盖）
>
> **修复方案**: PostgreSQL RPC 原子追加（预计 1-2 小时实施）

## 需求概述

为广告电商和漫剧创作者提供多角度批量图片生成功能。用户输入提示词 + 参考图后，系统通过 LLM 自动扩展提示词，一次生成 6-12 张覆盖多视角/多情境的图片。

## 核心设计决策

| 决策项 | 方案 |
|--------|------|
| 数据存储 | 一条记录，`image_urls` 数组存多张 |
| 积分计费 | N 张 × 单价 (2 credits/张) |
| Agent 标签 | 橙色/金色 (amber-500) |
| 提示扩展 | LLM (OpenAI 兼容 API，默认 Evolink) |
| 扩展提示可见性 | 默认隐藏 |
| 并发控制 | p-limit，最大 6 并发 |

### 2.1 提示词扩展服务

**文件**: `services/promptExpander.ts`

```typescript
import OpenAI from 'openai';

// 使用 OpenAI 兼容 API (支持 Evolink/OpenAI/Claude 等)
const llmClient = new OpenAI({
  apiKey: process.env.EVOLINK_API_KEY,
  baseURL: process.env.EVOLINK_BASE_URL || 'https://api.evolink.ai/v1',
});

export type UserContext = 'ecommerce' | 'comic' | 'general';

export async function expandImagePrompts(
  originalPrompt: string,
  imageCount: number,
  hasReferenceImages: boolean = false,
  userContext: UserContext = 'general'
): Promise<string[]> {
  // 构建系统提示词
  const systemPrompt = buildSystemPrompt(userContext);
  const userMessage = buildUserMessage(originalPrompt, imageCount, hasReferenceImages);

  const response = await llmClient.chat.completions.create({
    model: process.env.EVOLINK_MODEL || 'gemini-3-pro-preview',
    messages: [
      { role: 'system', content: systemPrompt },
      { role: 'user', content: userMessage },
    ],
    response_format: { type: 'json_object' },
  });

  return parseExpandedPrompts(response.choices[0].message.content, imageCount, originalPrompt);
}
```

### 2.2 Agent 图片生成服务

**文件**: `services/agentImageService.ts`

```typescript
import pLimit from 'p-limit';
import { expandImagePrompts, UserContext } from './promptExpander';
import { aiServiceManager } from './AIServiceManager';
import { createImageGeneration } from '@/models/imageGeneration';
import { decreaseCreditsMultiPool } from '@/services/credit';
import { calculateImageCredits } from '@/config/image-models';

const MAX_CONCURRENT = 6;

export async function generateAgentImages(
  params: AgentGenerationParams
): Promise<AgentGenerationResult> {
  // 1. 扩展提示词
  const expandedPrompts = await expandImagePrompts(
    prompt, imageCount, hasReferenceImages, userContext
  );

  // 2. 计算并扣除积分
  const singleImageCredits = calculateImageCredits(model, resolution);
  const totalCredits = singleImageCredits * imageCount;
  const creditDeduction = await decreaseCreditsMultiPool({...});

  // 3. 创建数据库记录
  const imageGeneration = await createImageGeneration({
    is_agent_mode: true,
    agent_image_count: imageCount,
    expanded_prompts: expandedPrompts,
    // ...
  });

  // 4. 并发生成图片 (最大 4 并发)
  const limit = pLimit(MAX_CONCURRENT);
  const tasks = expandedPrompts.map((expandedPrompt, idx) =>
    limit(async () => {
      const result = await aiServiceManager.generateImage(provider, {...});
      return { index: idx, success: true, taskId: result.taskId };
    })
  );

  const results = await Promise.all(tasks);

  return {
    generationId: imageGeneration.id,
    status: 'processing',
    imageCount,
    tasks: results,
  };
}
```

## 七、部分失败退款逻辑

采用**按比例退款**策略：成功 N 张扣 N 张积分，失败的自动退款。

```typescript
// 在回调处理中
async function handleAgentPartialRefund(
  imageGeneration: ImageGeneration,
  failedCount: number
): Promise<void> {
  const singleImageCredits = calculateImageCredits(
    imageGeneration.model_id,
    imageGeneration.metadata?.resolution
  );
  const refundAmount = singleImageCredits * failedCount;

  // 按原扣费池逆序退款
  const poolsReversed = [...creditDeduction.pools].reverse();
  for (const pool of poolsReversed) {
    const refundFromPool = Math.min(pool.deducted, remainingRefund);
    await increaseCredits({
      user_uuid: imageGeneration.user_id,
      trans_type: CreditsTransType.RefundImageGenerationFailed,
      credits: refundFromPool,
      order_no: pool.order_no,
    });
  }
}
```

## 十、并发回调问题分析与修复方案

### 10.1 问题描述

**Bug 现象**: Agent 模式生成 6 张图片时，如果 6 个回调在 1 秒内同时到达，最终数据库只保存最后 1 张图片，前面 5 张图片 URL 丢失。

**影响范围**:
- 用户支付了 12 积分（6 张 × 2 积分），但只收到 1 张图片
- 虽然退款逻辑会退还 5 张失败的积分（10 积分），但用户体验极差
- 发生概率: 100%（当 API 响应快时必现）

### 10.2 根因分析

#### 当前实现逻辑（有问题）

`app/api/ai-callback/[provider]/route.ts` 的 `handleAgentModeCallback` 函数：

```typescript
// Line 217-219: 读取最新记录
const latestRecord = (await getImageGenerationById(imageGeneration.id)) || imageGeneration;

// Line 237: 获取当前图片数组
const currentImageUrls = latestRecord.image_urls || [];

// Line 264-266: 追加新图片
const newImageUrls = Array.from(
  new Set([...currentImageUrls, newImageUrl])
);

// Line 326: 更新数据库（覆盖整个数组）
await updateImageGenerationById(imageGeneration.id, {
  image_urls: newImageUrls,  // ❌ 这里会覆盖其他回调的写入
  // ...
});
```

#### 并发场景时间线

```
时间 | Task1 回调 | Task2 回调 | Task3 回调 | 数据库状态
-----|-----------|-----------|-----------|-------------
T0   | 读取 (image_urls=[]) | | | image_urls=[]
T1   | | 读取 (image_urls=[]) | | image_urls=[]
T2   | | | 读取 (image_urls=[]) | image_urls=[]
T3   | 写入 [img1] | | | image_urls=[img1]
T4   | | 写入 [img2] | | image_urls=[img2] ❌ 覆盖了 img1
T5   | | | 写入 [img3] | image_urls=[img3] ❌ 覆盖了 img1,img2
```

**问题本质**: "读取-修改-写入"（Read-Modify-Write）操作不是原子的，多个并发请求会相互覆盖。

#### 为什么 Line 218 的"读取最新记录"无法解决问题

```typescript
const latestRecord = (await getImageGenerationById(imageGeneration.id)) || imageGeneration;
```

虽然每次都读取最新记录，但在**读取和写入之间没有锁**，其他回调可能已经修改了数据库。这是经典的 **Race Condition（竞态条件）**。

### 10.3 解决方案对比

| 方案 | 优点 | 缺点 | 实施成本 | 推荐 |
|------|------|------|---------|------|
| **PostgreSQL RPC 原子追加** | ✅ 原子性保证<br>✅ 零表结构改动<br>✅ 代码改动小 | ❌ 扩展性受限（12 张图以内） | ⭐⭐ 1-2 小时 | 🏆 **推荐** |
| **分表存储** | ✅ 架构清晰<br>✅ 扩展性好 | ❌ 改动大<br>❌ 需要数据迁移 | ⭐⭐⭐⭐ 4-6 小时 | 备选（3 个月后） |
| **Redis 分布式锁** | ✅ 可控性强 | ❌ 引入新依赖<br>❌ 运维复杂度高 | ⭐⭐⭐ 3-4 小时 | ❌ 不推荐 |
| **消息队列串行化** | ✅ 可靠性高 | ❌ 架构复杂<br>❌ 性能损失 | ⭐⭐⭐⭐⭐ 1 周+ | ❌ 不推荐 |

### 10.4 选定方案: PostgreSQL RPC 原子追加

#### 技术原理

利用 PostgreSQL 的 `array_append()` 函数，该函数在数据库层面是原子的：

```sql
-- 多个并发事务调用此函数时，PostgreSQL 会自动串行化
UPDATE image_generations
SET image_urls = array_append(image_urls, 'new_url')
WHERE id = 'xxx';
```

#### 架构优势

1. **原子性**: 数据库层面保证，无需应用层加锁
2. **最小改动**:
   - ✅ 零表结构改动
   - ✅ 零数据迁移
   - ✅ 前端无需改动
   - ✅ 只修改回调处理逻辑（约 50 行）
3. **性能好**: 直接在数据库操作，无额外依赖
4. **向后兼容**: 完全兼容现有单图模式

#### 实施步骤

##### Step 1: 创建 Supabase RPC 函数

在 Supabase Dashboard → SQL Editor 执行：

```sql
-- =====================================================
-- 原子追加图片 URL 到 Agent 生成记录
-- =====================================================
CREATE OR REPLACE FUNCTION append_image_to_agent_generation(
  p_generation_id UUID,
  p_image_url TEXT,
  p_r2_url TEXT DEFAULT NULL,
  p_task_id TEXT DEFAULT NULL,
  p_task_status TEXT DEFAULT 'completed'
)
RETURNS jsonb
LANGUAGE plpgsql
AS $$
DECLARE
  v_result jsonb;
  v_image_count INT;
  v_agent_count INT;
  v_failed_count INT;
  v_metadata jsonb;
BEGIN
  -- 原子更新：追加图片 URL 到数组
  UPDATE image_generations
  SET
    -- 原子追加到 image_urls 数组
    image_urls = array_append(COALESCE(image_urls, ARRAY[]::TEXT[]), p_image_url),

    -- 如果提供了 R2 URL，同样追加
    image_urls_r2 = CASE
      WHEN p_r2_url IS NOT NULL
      THEN array_append(COALESCE(image_urls_r2, ARRAY[]::TEXT[]), p_r2_url)
      ELSE image_urls_r2
    END,

    -- 更新图片计数
    image_count = array_length(array_append(COALESCE(image_urls, ARRAY[]::TEXT[]), p_image_url), 1),

    -- 追加任务状态到 metadata.agent_tasks 数组
    metadata = jsonb_set(
      COALESCE(metadata, '{}'::jsonb),
      '{agent_tasks}',
      COALESCE(metadata->'agent_tasks', '[]'::jsonb) ||
      jsonb_build_object(
        'taskId', p_task_id,
        'status', p_task_status,
        'imageUrl', p_image_url,
        'r2Url', p_r2_url,
        'completedAt', NOW()
      )
    ),

    updated_at = NOW()
  WHERE id = p_generation_id
  RETURNING
    image_count,
    agent_image_count,
    metadata
  INTO v_image_count, v_agent_count, v_metadata;

  -- 统计失败任务数量
  SELECT COUNT(*) INTO v_failed_count
  FROM jsonb_array_elements(v_metadata->'agent_tasks') t
  WHERE t->>'status' = 'failed';

  -- 返回结果
  v_result := jsonb_build_object(
    'generation_id', p_generation_id,
    'image_count', v_image_count,
    'agent_image_count', v_agent_count,
    'failed_count', v_failed_count,
    'all_done', (v_image_count + v_failed_count) >= v_agent_count
  );

  RETURN v_result;
END;
$$;

-- =====================================================
-- 标记 Agent 任务为失败
-- =====================================================
CREATE OR REPLACE FUNCTION mark_agent_task_failed(
  p_generation_id UUID,
  p_task_id TEXT,
  p_error TEXT DEFAULT 'Unknown error'
)
RETURNS jsonb
LANGUAGE plpgsql
AS $$
DECLARE
  v_result jsonb;
  v_image_count INT;
  v_agent_count INT;
  v_failed_count INT;
  v_metadata jsonb;
BEGIN
  -- 追加失败任务到 metadata.agent_tasks
  UPDATE image_generations
  SET
    metadata = jsonb_set(
      COALESCE(metadata, '{}'::jsonb),
      '{agent_tasks}',
      COALESCE(metadata->'agent_tasks', '[]'::jsonb) ||
      jsonb_build_object(
        'taskId', p_task_id,
        'status', 'failed',
        'error', p_error,
        'completedAt', NOW()
      )
    ),
    updated_at = NOW()
  WHERE id = p_generation_id
  RETURNING
    image_count,
    agent_image_count,
    metadata
  INTO v_image_count, v_agent_count, v_metadata;

  -- 统计失败任务数量
  SELECT COUNT(*) INTO v_failed_count
  FROM jsonb_array_elements(v_metadata->'agent_tasks') t
  WHERE t->>'status' = 'failed';

  -- 返回结果
  v_result := jsonb_build_object(
    'generation_id', p_generation_id,
    'image_count', COALESCE(v_image_count, 0),
    'agent_image_count', v_agent_count,
    'failed_count', v_failed_count,
    'all_done', (COALESCE(v_image_count, 0) + v_failed_count) >= v_agent_count
  );

  RETURN v_result;
END;
$$;
```

##### Step 2: 修改回调处理代码

**文件**: `app/api/ai-callback/[provider]/route.ts`

替换 `handleAgentModeCallback` 函数（line 212-406）：

```typescript
/**
 * 处理 Agent 模式的回调（使用 RPC 原子追加）
 * 聚合多张图片的结果到主记录
 */
async function handleAgentModeCallback(
  imageGeneration: ImageGeneration,
  processedResult: any,
  provider: AIServiceProvider
): Promise<Response> {
  const taskId = processedResult.taskId;
  const agentImageCount = imageGeneration.agent_image_count ||
                          imageGeneration.metadata?.agent_image_count || 0;

  console.log(`[Callback-Agent] Processing task ${taskId} for generation ${imageGeneration.id}`);

  if (processedResult.status === 'completed' && processedResult.images?.length > 0) {
    // ============ 图片生成成功 ============
    const newImageUrl = processedResult.images[0].url;
    console.log(`[Callback-Agent] Image received: ${newImageUrl}`);

    // 上传到 R2
    let r2Url: string | null = null;
    try {
      const uploadResult = await ImageStorageService.uploadImagesToR2(
        processedResult.images,
        `${imageGeneration.id}-${Date.now()}`
      );
      if (uploadResult.success && uploadResult.r2Urls?.length > 0) {
        r2Url = uploadResult.r2Urls[0];
        console.log(`[Callback-Agent] R2 uploaded: ${r2Url}`);
      }
    } catch (uploadError) {
      console.error(`[Callback-Agent] R2 upload failed:`, uploadError);
    }

    // 🔥 使用 RPC 原子追加图片
    const { data, error } = await getSupabaseClient()
      .rpc('append_image_to_agent_generation', {
        p_generation_id: imageGeneration.id,
        p_image_url: newImageUrl,
        p_r2_url: r2Url,
        p_task_id: taskId,
        p_task_status: 'completed',
      });

    if (error) {
      console.error(`[Callback-Agent] RPC append failed:`, error);
      return respErr(`Failed to append image: ${error.message}`);
    }

    const result = data as {
      generation_id: string;
      image_count: number;
      agent_image_count: number;
      failed_count: number;
      all_done: boolean
    };

    console.log(`[Callback-Agent] Image appended successfully. Progress: ${result.image_count}/${result.agent_image_count}, Failed: ${result.failed_count}`);

    // 检查是否全部完成
    if (result.all_done) {
      console.log(`[Callback-Agent] All tasks done. Final count: ${result.image_count}/${result.agent_image_count}, Failed: ${result.failed_count}`);

      const updateData: any = {
        status: 'COMPLETED',
        completed_at: new Date().toISOString(),
      };

      // 部分失败退款
      if (result.failed_count > 0) {
        await handleAgentPartialRefund(imageGeneration, result.failed_count);
        const singleImageCredits = calculateImageCredits(
          imageGeneration.model_id,
          imageGeneration.metadata?.resolution
        );
        updateData.credits_used = imageGeneration.credits_used - (singleImageCredits * result.failed_count);
        console.log(`[Callback-Agent] Refunded ${singleImageCredits * result.failed_count} credits for ${result.failed_count} failed tasks`);
      }

      await updateImageGenerationById(imageGeneration.id, updateData);
      console.log(`[Callback-Agent] Generation ${imageGeneration.id} marked as COMPLETED`);
    }

    return respData({
      success: true,
      message: "Agent callback processed",
      taskId,
      generationId: imageGeneration.id,
      completedCount: result.image_count,
      failedCount: result.failed_count,
      totalCount: result.agent_image_count,
      allDone: result.all_done,
    });

  } else if (processedResult.status === 'failed') {
    // ============ 单个任务失败 ============
    console.error(`[Callback-Agent] Task ${taskId} failed: ${processedResult.error}`);

    // 🔥 使用 RPC 标记任务失败
    const { data, error } = await getSupabaseClient()
      .rpc('mark_agent_task_failed', {
        p_generation_id: imageGeneration.id,
        p_task_id: taskId,
        p_error: processedResult.error || 'Unknown error',
      });

    if (error) {
      console.error(`[Callback-Agent] Mark failed RPC error:`, error);
      return respErr(`Failed to mark task failed: ${error.message}`);
    }

    const result = data as {
      generation_id: string;
      image_count: number;
      agent_image_count: number;
      failed_count: number;
      all_done: boolean
    };

    console.log(`[Callback-Agent] Task marked as failed. Progress: ${result.image_count}/${result.agent_image_count}, Failed: ${result.failed_count}`);

    // 检查是否全部完成
    if (result.all_done) {
      console.log(`[Callback-Agent] All tasks done (with failures). Final count: ${result.image_count}/${result.agent_image_count}, Failed: ${result.failed_count}`);

      if (result.image_count === 0) {
        // 全部失败
        await updateImageGenerationById(imageGeneration.id, {
          status: 'FAILED',
          error_message: 'All agent tasks failed',
        });
        await handleAgentFullRefund(imageGeneration);
        console.log(`[Callback-Agent] Generation ${imageGeneration.id} marked as FAILED (all tasks failed)`);
      } else {
        // 部分成功
        const updateData: any = {
          status: 'COMPLETED',
          completed_at: new Date().toISOString(),
        };
        await handleAgentPartialRefund(imageGeneration, result.failed_count);
        const singleImageCredits = calculateImageCredits(
          imageGeneration.model_id,
          imageGeneration.metadata?.resolution
        );
        updateData.credits_used = imageGeneration.credits_used - (singleImageCredits * result.failed_count);
        await updateImageGenerationById(imageGeneration.id, updateData);
        console.log(`[Callback-Agent] Generation ${imageGeneration.id} marked as COMPLETED (partial success)`);
      }
    }

    return respData({
      success: true,
      message: "Agent callback processed (task failed)",
      taskId,
      generationId: imageGeneration.id,
      completedCount: result.image_count,
      failedCount: result.failed_count,
      totalCount: result.agent_image_count,
      allDone: result.all_done,
    });
  }

  // ============ 其他状态（processing 等）============
  console.log(`[Callback-Agent] Task ${taskId} status: ${processedResult.status} (no action needed)`);
  return respData({
    success: true,
    message: "Agent callback received",
    taskId,
    status: processedResult.status,
  });
}
```
