## 详细流程图

### 1. 普通模式提交流程

```mermaid
sequenceDiagram
    participant User as 用户/前端
    participant Submit as /api/image-generation/submit
    participant Auth as NextAuth
    participant Credit as creditService
    participant AIS as AIServiceManager
    participant Provider as AI提供商
    participant DB as Supabase

    User->>Submit: POST 请求<br/>{model, prompt, mode, provider...}
    Submit->>Auth: auth() 验证登录
    Auth-->>Submit: session + userId

    Submit->>DB: getUserInfo(userId)
    DB-->>Submit: userInfo

    Submit->>Credit: getUserCredits(userId)
    Credit-->>Submit: 当前积分余额

    alt 积分 <= 12 (新用户)
        Submit->>Submit: verifyCaptcha(token)
    end

    Submit->>AIS: getProvider(provider)
    AIS-->>Submit: providerInstance

    Submit->>Submit: calculateImageCredits(model, resolution)
    Note over Submit: 计算需要扣除的积分

    Submit->>Credit: decreaseCredits(userId, credits)
    Credit-->>Submit: deductResult<br/>{pools, totalDeducted}
    Note over Submit: 记录扣费详情到 metadata.credit_deduction

    Submit->>DB: createImageGeneration()<br/>status: IN_QUEUE
    DB-->>Submit: generationId

    Submit->>AIS: generateImage(params)
    AIS->>Provider: 调用AI API

    alt 同步返回 (如 DALL-E)
        Provider-->>AIS: {status: completed, images: [...]}
        AIS-->>Submit: result
        Submit->>DB: updateImageGenerationById()<br/>status: COMPLETED
        Submit-->>User: {code: 0, data: {id, status, image_urls}}
    else 异步回调 (如 Nano Banana)
        Provider-->>AIS: {status: pending, taskId: xxx}
        AIS-->>Submit: result
        Submit-->>User: {code: 0, data: {id, status: in_queue, task_id}}
        Note over User: 前端开始轮询或等待回调
    else 错误
        Provider-->>AIS: error
        AIS-->>Submit: error
        Submit->>Credit: increaseCredits(退款)
        Submit->>DB: updateImageGenerationById()<br/>status: FAILED
        Submit-->>User: {code: -1, message: error}
    end
```

### 2. Agent 模式批量生成流程

```mermaid
sequenceDiagram
    participant User as 用户/前端
    participant Submit as /api/image-generation/submit
    participant Agent as agentImageService
    participant Expand as promptExpander
    participant Credit as creditService
    participant AIS as AIServiceManager
    participant Provider as AI提供商
    participant DB as Supabase

    User->>Submit: POST 请求<br/>{agent_mode: true, agent_image_count: 6...}

    Submit->>Agent: generateAgentImages(params)

    Agent->>Expand: expandImagePrompts(prompt, count)
    Note over Expand: Claude AI 分析提示词<br/>生成多角度子提示词
    Expand-->>Agent: expandedPrompts[6]

    Agent->>Agent: calculateImageCredits() * 6
    Note over Agent: 计算总积分需求

    Agent->>Credit: decreaseCredits(总积分)
    Credit-->>Agent: deductResult

    Agent->>DB: createImageGeneration()<br/>metadata: {<br/>  is_agent_mode: true,<br/>  agent_image_count: 6,<br/>  expanded_prompts: [...],<br/>  agent_task_ids: [],<br/>  credit_deduction: {...}<br/>}
    DB-->>Agent: generationId

    par 并发生成 (pLimit=6)
        Agent->>AIS: generateImage(expandedPrompts[0])
        AIS->>Provider: API调用
        Provider-->>AIS: taskId_0
        AIS-->>Agent: result_0
    and
        Agent->>AIS: generateImage(expandedPrompts[1])
        AIS->>Provider: API调用
        Provider-->>AIS: taskId_1
        AIS-->>Agent: result_1
    and
        Agent->>AIS: generateImage(expandedPrompts[2])
        Note over Agent: ... 继续并发 3-5
    end

    Agent->>DB: updateImageGenerationById()<br/>metadata.agent_task_ids = [taskId_0, taskId_1, ...]

    Agent-->>Submit: {id, tasks: [...], is_agent_mode: true}
    Submit-->>User: {code: 0, data: {...}}
```

### 3. 异步回调处理流程

```mermaid
sequenceDiagram
    participant Provider as AI提供商
    participant Callback as /api/ai-callback/:provider
    participant AIS as AIServiceManager
    participant DB as Supabase
    participant R2 as imageStorageService
    participant Credit as creditService

    Provider->>Callback: POST 回调<br/>{taskId, state, result: [{url}]}

    Callback->>AIS: getProvider(provider)
    AIS-->>Callback: providerInstance

    Callback->>Callback: provider.handleCallback(payload)
    Note over Callback: 解析提供商特定格式<br/>转换为标准 ProviderResponse

    alt 普通模式
        Callback->>DB: getImageGenerationByProviderTaskId(taskId)
        DB-->>Callback: generation

        alt 成功
            Callback->>R2: uploadImagesToR2(urls)
            R2-->>Callback: r2Urls

            Callback->>DB: updateImageGenerationById()<br/>status: COMPLETED<br/>image_urls: [...原始URL]<br/>image_urls_r2: [...R2 URL]
        else 失败
            Callback->>Credit: increaseCredits(退款)
            Callback->>DB: updateImageGenerationById()<br/>status: FAILED
        end

    else Agent 模式
        Callback->>DB: getImageGenerationByAgentTaskId(taskId)
        Note over DB: 搜索 metadata.agent_task_ids 数组
        DB-->>Callback: generation + taskIndex

        Callback->>Callback: 更新 metadata.agent_tasks[index]

        Callback->>R2: uploadImagesToR2([新图片])
        R2-->>Callback: r2Urls

        Callback->>Callback: 追加到 image_urls 数组

        Callback->>Callback: 检查所有任务完成状态

        alt 全部完成
            Callback->>DB: updateImageGenerationById()<br/>status: COMPLETED
        else 部分完成
            Callback->>DB: updateImageGenerationById()<br/>status: IN_PROGRESS
        else 全部失败
            Callback->>Credit: increaseCredits(全额退款)
            Callback->>DB: updateImageGenerationById()<br/>status: FAILED
        end
    end

    Callback-->>Provider: {success: true}
```

### 4. 前端轮询流程

```mermaid
sequenceDiagram
    participant UI as 前端组件
    participant Poll as imagePollingService
    participant Status as /api/image-generation/status
    participant AIS as AIServiceManager
    participant DB as Supabase

    UI->>Poll: startPolling(generationIds)

    loop 每3秒轮询
        Poll->>Status: POST {id}
        Status->>DB: getImageGenerationById(id)
        DB-->>Status: generation

        alt 状态未完成
            Status->>AIS: getTaskStatus(taskId)
            Note over AIS: 某些提供商支持轮询<br/>Nano Banana 不支持

            alt 提供商返回新状态
                AIS-->>Status: newStatus
                Status->>DB: updateImageGenerationById()
            end
        end

        Status-->>Poll: {id, status, image_urls, ...}
        Poll-->>UI: 更新状态

        alt status === "completed" || status === "failed"
            Poll->>Poll: stopPolling(id)
        end
    end
```

### 5. 历史记录查询流程

```mermaid
sequenceDiagram
    participant UI as 前端页面
    participant History as /api/image-generations/history
    participant DB as Supabase

    UI->>History: GET ?page=1&limit=20

    History->>History: auth() 验证

    History->>DB: getUserImageGenerations()<br/>WHERE user_id = ? AND is_delete = false<br/>ORDER BY created_at DESC<br/>LIMIT 20 OFFSET 0

    DB-->>History: {data: [...], total: 145}

    loop 格式化每条记录
        History->>History: 转换字段格式
        Note over History: - status 转小写<br/>- 提取 image_url (第一张)<br/>- 推算 resolution<br/>- 提取 Agent 模式字段
    end

    History-->>UI: {<br/>  code: 0,<br/>  data: [...],<br/>  pagination: {page, limit, total, totalPages}<br/>}

    UI->>UI: 识别未完成的任务

    alt 存在未完成任务
        UI->>UI: imagePollingService.startPolling()
    end
```

## 积分扣费与退款逻辑

```mermaid
flowchart TB
    subgraph 扣费流程
        A[计算所需积分] --> B[检查余额]
        B --> |余额足够| C[decreaseCredits]
        B --> |余额不足| D[返回错误]
        C --> E[记录扣费详情]
        E --> F["metadata.credit_deduction = {<br/>pools: [{deducted, order_no, expired_at}],<br/>total_deducted,<br/>deducted_at}"]
    end

    subgraph 退款流程
        G[生成失败] --> H{检查扣费记录}
        H --> |有记录| I[遍历 pools]
        H --> |无记录| J[跳过退款]
        I --> K["increaseCredits({<br/>credits: pool.deducted,<br/>order_no: pool.order_no,<br/>expired_at: pool.expired_at})"]
        K --> L[退回原池并保留过期时间]
    end

    subgraph Agent模式部分退款
        M[部分任务失败] --> N[计算失败比例]
        N --> O["failedCount / totalCount"]
        O --> P[按比例从各池退款]
    end
```

## 状态机流转

```mermaid
stateDiagram-v2
    [*] --> PENDING: 创建记录
    PENDING --> IN_QUEUE: 提交到AI服务
    IN_QUEUE --> IN_PROGRESS: 开始处理
    IN_PROGRESS --> COMPLETED: 生成成功
    IN_PROGRESS --> FAILED: 生成失败
    IN_QUEUE --> FAILED: 提交失败
    PENDING --> FAILED: 验证失败
    COMPLETED --> [*]
    FAILED --> [*]

    note right of PENDING: 刚创建,尚未提交
    note right of IN_QUEUE: 已提交,等待处理
    note right of IN_PROGRESS: 正在生成中
    note right of COMPLETED: 生成成功,有图片URL
    note right of FAILED: 失败,已退款
```

### Agent 模式子任务状态

```mermaid
stateDiagram-v2
    direction LR

    [*] --> submitted: 提交任务
    submitted --> processing: 回调更新
    processing --> completed: 成功
    processing --> failed: 失败
    submitted --> failed: 超时

    state "主记录状态判断" as check {
        [*] --> 检查所有子任务
        检查所有子任务 --> 全部completed: 主记录COMPLETED
        检查所有子任务 --> 全部failed: 主记录FAILED
        检查所有子任务 --> 部分完成: 主记录IN_PROGRESS
    }
```
