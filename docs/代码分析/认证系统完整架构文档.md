## 系统概述

Veo3 AI 平台的认证系统采用分层架构设计，整合了 NextAuth v5 和 Supabase Auth，提供多种登录方式和完善的用户管理功能。系统设计遵循关注点分离原则，将认证逻辑、会话管理和用户数据管理清晰分离。

### 主要特性
- 多种认证方式（邮箱密码、Google、GitHub、Apple、Google One Tap）
- 邮箱验证机制
- 密码重置功能
- 会话管理
- 新用户自动赠送积分
- 邀请码系统
- 管理员权限控制

## 技术栈

- **前端框架**: Next.js 14 (App Router)
- **认证库**: NextAuth v5
- **后端服务**: Supabase Auth
- **数据库**: PostgreSQL (通过 Supabase)
- **状态管理**: React Context API
- **表单处理**: React Hook Form
- **类型安全**: TypeScript
- **验证库**: Zod

## 认证方式

### 1. 邮箱密码认证
- 使用 Supabase Auth 服务
- 支持邮箱验证
- 密码重置功能

### 2. OAuth 认证
- Google OAuth
- GitHub OAuth
- Apple OAuth
- Google One Tap（快速登录）

### 邮箱密码登录完整流程

#### 1. 用户输入凭据（前端）
**文件**: `/components/sign/form.tsx`

用户在登录表单中输入邮箱和密码，点击提交按钮触发登录流程：

```typescript
const onSubmit = async (data: EmailFormData) => {
  // 清除之前的错误信息
  clearError();
  setMessage(null);

    }
  }

```

#### 2. Hook 层处理（客户端）
**文件**: `/hooks/useEmailAuth.ts`

Hook 负责管理登录状态和调用 API：

```typescript
const login = async (data: LoginData, callbackUrl?: string) => {
  setIsLoading(true);
  setError(null);

  try {
    // 第一步：调用后端 API 验证凭据
    const response = await fetch("/api/auth/email-signin", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify(data),
    });

    const result = await response.json();

    if (!response.ok) {
      // 处理邮箱未验证错误
      if (result.error === "email_not_confirmed") {
        return { error: "EMAIL_NOT_CONFIRMED", email: result.email };
      }
      setError(result.message);
      return false;
    }

    // 第二步：凭据验证成功后，使用 NextAuth 创建会话
    const authResult = await signIn("email", {
      email: data.email,
      password: data.password,
      redirect: false,
    });

    if (authResult?.ok) {
      // 登录成功，重定向到目标页面
      router.push(callbackUrl || "/home");
      return true;
    }
  } catch (error) {
    setError("Login failed. Please try again.");
    return false;
  }
};
```

#### 3. API 路由验证（服务端）
**文件**: `/app/api/auth/email-signin/route.ts`

API 路由负责验证用户凭据：

```typescript
export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    
    // 使用 Zod 验证输入数据
    const validation = signinSchema.safeParse(body);
    if (!validation.success) {
      return NextResponse.json(
        { error: validation.error.errors[0].message },
        { status: 400 }
      );
    }

    const { email, password } = validation.data;

    // 调用 Supabase 认证服务
    const user = await signInWithEmail(email, password);
    
    // 返回成功响应
    return NextResponse.json({
      success: true,
      user: {
        id: user.id,
        email: user.email,
        name: user.name,
      }
    });

  } catch (error: any) {
    // 处理特殊错误类型
    if (error.message === "EMAIL_NOT_CONFIRMED") {
      return NextResponse.json(
        { 
          error: "email_not_confirmed",
          message: "Please verify your email first",
          email: email
        },
        { status: 403 }
      );
    }
    
    // 处理其他错误
    return NextResponse.json(
      { 
        error: "invalid_credentials",
        message: error.message
      },
      { status: 401 }
    );
  }
}
```

#### 4. Supabase 认证服务
**文件**: `/services/supabase-auth.ts`

与 Supabase Auth 服务交互：

```typescript
export async function signInWithEmail(email: string, password: string) {
  try {
    // 创建 Supabase 客户端
    const supabase = getSupabaseAuthClient();

    // 调用 Supabase Auth API
    const { data, error } = await supabase.auth.signInWithPassword({
      email,
      password,
    });

    if (error) {
      // 处理各种错误情况
      if (error.message?.includes("Email not confirmed")) {
        throw new Error("EMAIL_NOT_CONFIRMED");
      }
      
      if (error.message?.includes("rate limit")) {
        throw new Error("Too many login attempts");
      }
      
      throw new Error("Invalid email or password");
    }

    // 返回符合 NextAuth 格式的用户数据
    return {
      id: data.user.id,
      email: data.user.email || "",
      name: data.user.user_metadata?.name || data.user.email?.split("@")[0] || "",
      image: data.user.user_metadata?.avatar_url || "",
      emailVerified: data.user.email_confirmed_at 
        ? new Date(data.user.email_confirmed_at) 
        : null,
    };
  } catch (error) {
    throw error;
  }
}
```

#### 5. NextAuth 会话创建
**文件**: `/auth/config.ts`

NextAuth 处理会话创建和用户数据持久化：

```typescript
// Email Provider 配置
CredentialsProvider({
  id: "email",
  name: "Email",
  credentials: {
    email: { label: "Email", type: "email" },
    password: { label: "Password", type: "password" },
  },
  async authorize(credentials, req) {
    if (!credentials?.email || !credentials?.password) {
      return null;
    }

    try {
      // 再次调用 Supabase 认证（双重验证）
      const user = await signInWithEmail(
        credentials.email as string,
        credentials.password as string
      );
      return user;
    } catch (error) {
      return null;
    }
  },
})
```

#### 6. JWT 回调 - 用户数据持久化
**文件**: `/auth/config.ts`

JWT 回调负责在数据库中创建或更新用户记录：

```typescript
async jwt({ token, user, account }) {
  try {
    if (user && user.email && account) {
      // 构建用户数据对象
      const dbUser: User = {
        uuid: getUuid(),                    // 生成唯一ID
        email: user.email,
        nickname: user.name || user.email.split("@")[0] || "",
        avatar_url: user.image || getDefaultAvatar(user.email),
        signin_type: account.type,          // "credentials"
        signin_provider: account.provider,   // "email"
        signin_openid: account.provider === "email" 
          ? user.id 
          : account.providerAccountId,
        created_at: getIsoTimestr(),
        signin_ip: await getClientIp(),
      };

      try {
        // 保存用户到数据库
        const savedUser = await saveUser(dbUser);

        // 将用户信息存储到 JWT token 中
        token.user = {
          uuid: savedUser.uuid,
          email: savedUser.email,
          nickname: savedUser.nickname,
          avatar_url: savedUser.avatar_url,
          created_at: savedUser.created_at,
        };
      } catch (e) {
        console.error("save user failed:", e);
      }
    }
    return token;
  } catch (e) {
    console.error("jwt callback error:", e);
    return token;
  }
}
```

#### 7. 用户服务层
**文件**: `/services/user.ts`

处理用户数据的业务逻辑：

```typescript
export async function saveUser(user: User) {
  try {
    // 检查用户是否已存在
    const existUser = await findUserByEmail(user.email);
    
    if (!existUser) {
      // 新用户处理流程
      await insertUser(user);

      // 生成并保存邀请码
      const inviteCode = generateInviteCode();
      if (user.uuid) {
        await updateUserInviteCode(user.uuid, inviteCode);
        user.invite_code = inviteCode;
      }

      // 新用户赠送积分（1个月后过期）
      await increaseCredits({
        user_uuid: user.uuid || "",
        trans_type: CreditsTransType.NewUser,
        credits: CreditsAmount.NewUserGet,
        expired_at: getOneMonthLaterTimestr(),
      });
    } else {
      // 已存在用户，使用现有数据
      user.id = existUser.id;
      user.uuid = existUser.uuid;
      user.created_at = existUser.created_at;
    }

    return user;
  } catch (e) {
    console.log("save user failed: ", e);
    throw e;
  }
}
```

#### 8. 数据库操作层
**文件**: `/models/user.ts`

直接与数据库交互的函数：

```typescript
// 插入新用户
export async function insertUser(user: User) {
  const supabase = getSupabaseClient();
  const { data, error } = await supabase
    .from("users")
    .insert(user);

  if (error) {
    throw error;
  }
  return data;
}

// 根据邮箱查找用户
export async function findUserByEmail(email: string): Promise<User | undefined> {
  const supabase = getSupabaseClient();
  const { data, error } = await supabase
    .from("users")
    .select("*")
    .eq("email", email)
    .limit(1)
    .single();

  if (error) {
    return undefined;
  }
  return data;
}

// 更新用户邀请码
export async function updateUserInviteCode(
  user_uuid: string,
  invite_code: string
) {
  const supabase = getSupabaseClient();
  const updated_at = getIsoTimestr();
  const { data, error } = await supabase
    .from("users")
    .update({ invite_code, updated_at })
    .eq("uuid", user_uuid);

  if (error) {
    throw error;
  }
  return data;
}
```

export const AppContextProvider = ({ children }: { children: ReactNode }) => {

    }
  }, [session]);

### OAuth 登录流程

OAuth 登录流程相对简单，主要由 NextAuth 处理：

1. 用户点击 OAuth 登录按钮
2. NextAuth 重定向到第三方认证页面
3. 用户授权后，第三方服务重定向回应用
4. NextAuth 处理回调，获取用户信息
5. JWT 回调保存/更新用户数据（流程同上）
6. 创建会话，用户登录成功

## 数据库设计

系统使用两个主要的数据表：

### 1. Supabase Auth 表（auth.users）
- Supabase 内置的认证表
- 存储认证凭据和基本信息
- 由 Supabase Auth 服务管理

### 2. 应用用户表（public.users）
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  uuid VARCHAR(255) UNIQUE NOT NULL,
  email VARCHAR(255) UNIQUE NOT NULL,
  nickname VARCHAR(255),
  avatar_url TEXT,
  signin_type VARCHAR(50),        -- credentials, oauth
  signin_provider VARCHAR(50),    -- email, google, github, apple
  signin_openid VARCHAR(255),     -- 第三方平台的用户ID
  invite_code VARCHAR(50) UNIQUE,
  invited_by VARCHAR(255),        -- 邀请人的 UUID
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  signin_ip VARCHAR(50)
);
```

### 3. 积分表（credits_transactions）
```sql
CREATE TABLE credits_transactions (
  id SERIAL PRIMARY KEY,
  user_uuid VARCHAR(255) NOT NULL,
  trans_type VARCHAR(50) NOT NULL,  -- NewUser, Purchase, Usage, etc.
  credits INTEGER NOT NULL,
  expired_at TIMESTAMP,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 安全机制

### 1. 密码安全
- 密码由 Supabase Auth 加密存储
- 使用 bcrypt 或类似算法
- 密码重置通过邮件验证

### 2. 会话安全
- 使用 JWT 存储会话信息
- Token 定期刷新
- HttpOnly Cookie 存储

### 3. 输入验证
- 使用 Zod 进行服务端输入验证
- 防止 SQL 注入
- XSS 防护

### 4. 速率限制
- Supabase Auth 内置速率限制
- 防止暴力破解攻击

### 5. 邮箱验证
- 新用户必须验证邮箱
- 未验证用户无法登录

## 错误处理

系统对不同类型的错误有专门的处理机制：

### 1. 邮箱未验证
- 错误代码：`EMAIL_NOT_CONFIRMED`
- 用户体验：显示验证提示，提供重新发送验证邮件选项

### 2. 凭据错误
- 错误代码：`invalid_credentials`
- 用户体验：显示通用错误信息，避免泄露账号存在性

### 3. 速率限制
- 错误代码：`too_many_requests`
- 用户体验：提示用户稍后重试

### 4. 网络错误
- 通过 try-catch 捕获
- 显示友好的错误提示

## 总结

Veo3 AI 的认证系统通过分层架构实现了清晰的关注点分离：

1. **表现层**：React 组件和 Hooks 处理用户交互
2. **API 层**：Next.js API Routes 提供后端接口
3. **认证层**：NextAuth 管理会话和认证流程
4. **服务层**：封装业务逻辑
5. **数据层**：处理数据库操作

这种架构设计使得系统易于维护和扩展，同时提供了良好的用户体验和安全保障。通过整合 NextAuth 和 Supabase Auth，系统既获得了成熟的认证解决方案，又保持了灵活性和可定制性。
