# Sora2 模型会员权限需求

## 背景
现有视频生成模块支持多模型切换，需要在前端对 `sora2` 模型进行会员专属控制。

## 用户与前提
- 会员：在前端上下文中被识别为有效会员。
- 非会员：无会员权限或状态未知时默认视为非会员。

## 功能要求
- 仅会员用户可以选择 `sora-2-text-to-video` 与 `sora-2-image-to-video` 模型。
- 非会员用户不可切换至上述模型，保持原模型选中状态。
- 会员用户选择 `sora2` 后沿用当前生成流程，无额外提示。

## UI
- 在下拉列表中为 `sora2` 模型标题右侧展示皇冠标识，样式与水印开关使用的皇冠图标保持一致（颜色、尺寸、间距）。
- 非会员视角下 `sora2` 选项使用禁用态视觉（降低不透明度、光标不可用）。

## 交互
- 非会员点击 `sora2` 选项时阻止选中，并通过 Toast 告知“仅会员可使用该模型”。
- 会员点击 `sora2` 选项不触发额外提示。

## 状态校验
- 模块初始化时依据前端会员态决定 `sora2` 可交互性；当会员状态尚未确认时按非会员处理。

## 实现说明
- 在 `config/video-models.ts` 中为需会员权限的模型配置 `requiresMembership: true`，前端会自动展示皇冠标识并限制非会员选择。
