## 三、封禁理由与判定标准

### 批量注册 IP 判定标准

1. **数量阈值**: 单 IP 在 7 天内注册 ≥ 5 个账户
2. **时间模式**: 注册时间过于集中（如34小时12个账户）
3. **邮箱模式**:
   - Gmail "+" 别名批量生成
   - 随机字符串邮箱
   - 集中使用临时邮箱域名
4. **行为特征**: 注册后立即耗尽免费额度，无付费意图

### 一次性邮箱判定标准

1. **已知临时邮箱服务**:
   - yopmail.com, trashlify.com, sharklasers.com（公开服务）
   - webxios.pro, quiet-branch.com（专业临时邮箱）

2. **域名特征**:
   - 含有 temp, mail, disposable, trash 关键词
   - 使用生僻 TLD（.pro, .live, .cfd, .ovh）
   - 域名注册时间极短

3. **使用模式**:
   - 多个 IP 集中使用同一域名
   - 或单个 IP 批量使用（异常集中）

4. **IP/账户比异常**:
   - 接近 1.0 → 高度分布式攻击
   - < 0.5 → 集中式滥用

### 不封禁的例外情况

- **付费用户**: 如 `haomarales@gmail.com`（12笔订单，$273总付费）
- **正常邮箱服务**: gmail.com, mail.ru, yandex.ru, outlook.com, yahoo.com
  - 虽有批量注册现象，但混有大量正常用户
  - 需通过 IP 维度过滤，而非邮箱域名
- **企业邮箱**: 需人工审核确认

## 五、应用层防护建议

### 1. 邮箱域名黑名单（注册时验证）

**文件**: `lib/validators/email.ts`

```typescript
// 一次性邮箱黑名单
export const DISPOSABLE_EMAIL_DOMAINS = [
  'webxios.pro', 'yopmail.com', 'quiet-branch.com', 'radiant-flow.org',

  'moonfee.com', 'sharklasers.com', 'dunefee.com', 'picoaxis.com',
  // ... 完整列表见上文
];

export function isDisposableEmail(email: string): boolean {
  const domain = email.split('@')[1]?.toLowerCase();
  return domain ? DISPOSABLE_EMAIL_DOMAINS.includes(domain) : false;
}
```

### 2. IP 限流（Nginx/Cloudflare）

**Nginx 配置示例**:

```nginx
# 限制单 IP 注册频率
limit_req_zone $binary_remote_addr zone=register_limit:10m rate=3r/w;

  limit_req zone=register_limit burst=1 nodelay;
  # ...

```

**Cloudflare WAF 规则**:
- 单 IP 24小时内超过 3 次注册 → 强制 CAPTCHA
- 单 IP 1小时内超过 5 次 API 调用 → 限流

## 六、后续监控建议

### 每日监控指标

1. **新注册用户的 IP 分布**
   - 单 IP 注册数 > 3 → 自动告警

2. **新出现的邮箱域名**
   - 新域名注册数 > 5 → 人工审核

3. **封禁账户尝试登录统计**
   - 监控封禁后的访问尝试

### 每周审查

1. 生成封禁效果报告
2. 更新黑名单（IP + 邮箱域名）
3. 评估误封风险

### 告警阈值

| 指标 | 阈值 | 告警级别 |

| 新临时邮箱域名账户数 | ≥ 5 | 中 |
