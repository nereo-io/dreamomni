# Veo3 Reference-to-Video（R2V）功能调研（用户视角）

> 更新时间：2025-10-21（美东）

## 1. 功能定义

**一句话**：上传 1–3 张参考图，Veo 3.1 生成 8 秒视频并尽量保持主体外观一致，可带原生音频，支持 16:9 与 9:16。 citeturn3view0turn4view0

**详细说明**

- **核心机制**：把参考图作为“资产参考”（asset），用于锁定人物/物体/产品外观，视频内尽量保持一致。非风格迁移。 citeturn4view0
- **与 Image-to-Video 的区别**：I2V 是“拿一张图做起始帧并动画化”；R2V 不以某张图为起始帧，而是把 1–3 张图当“约束”，生成全新片段。 citeturn3view0
- **技术原理（简版）**：模型接收参考图并提取主体特征向量，在生成过程中对齐这些特征以维持外观一致；官方表述为“preserve the subject's appearance”。 citeturn3view0

## 2. 核心卖点（用户价值）

卖点 1：跨镜头角色一致性
用户价值：在多场景叙事中保持角色外观一致，是专业视频制作的标志，也能强化品牌形象
技术支撑：AI 维护主体的身份和外观，确保视频所有帧的完美连续性
卖点 2：原生音频同步
用户价值：省去后期配音成本，对话、环境音、音效自动匹配画面
技术支撑：Veo 3.1 带来更丰富、自然的声音，与屏幕动作同步性更好，增强整体真实感
卖点 3：多场景叙事能力
用户价值：通过多提示技术实现复杂叙事，用最少输入设计多样场景，同时保持角色身份和叙事连贯性
技术支撑：视频扩展和场景连续性功能，通过 Gemini API 和 Flow 的集成工作流突破 8 秒限制
卖点 4：精细化控制
用户价值：控制首尾帧、相机运动、对象位置，实现电影级精准
技术支撑：首尾帧功能在提供的起始和结束图像之间生成自然过渡，配备音频

## 3. 使用流程（3–4 步）

**Step 1 ｜准备参考图**

- **操作**：挑 1–3 张同一主体清晰图。
- **注意**：JPEG/PNG；建议 720p 或更高，构图简单、光照统一更稳。
- **最佳实践**：侧面/正面/半身搭配，提高模型识别稳定性。

**Step 2 ｜在控制台/接口选择 R2V**

- **操作**：Vertex AI Studio 选 Veo 3.1，Reference 选 “Subject”，上传 1–3 张。
- **注意**：使用参考图时，当前返回长度为 8 秒。
- **最佳实践**：Prompt 写清镜头、动作、场景、氛围与音效。

**Step 3 ｜生成与复用**

- **操作**：生成 8 秒片段；若需更长，用 Extend 串接。
- **注意**：Extend 仅支持“Veo 生成的视频”继续延长。
- **最佳实践**：每次延展在末 1 秒给出承接动作/音效提示。

## 4. 技术要求

- **参考图格式**：JPEG、PNG。 citeturn4view0
- **尺寸建议**：≥720p，16:9 或 9:16 更稳（非常规比例可能被裁/缩放）。 citeturn5search5
- **数量**：1–3 张；多张可覆盖不同视角/服饰细节。 citeturn3view0
- **时长**：R2V 基础生成 8 秒；可用 Extend 逐步加长。 citeturn4view0turn3view0
- **分辨率/比例**：生成支持 720p/1080p，16:9 或 9:16。 citeturn1view0turn9view0
- **费用（开发者参考）**：官方 API 定价：Veo 3 $0.40/秒、Veo 3 Fast $0.15/秒；第三方托管（如 fal.ai）价格可能不同。 citeturn9view0turn8search0turn8search2

> 备注：Veo 3.1 不支持“style reference”（风格参考），风格图仅在 Veo 2 有效。 citeturn4view0

## 5. 常见问题（FAQ）

1. **能 100% 锁脸/锁造型吗？**  
   **答**：目标是“尽量保持主体外观”，非 100% 保证；多角度清晰图更稳。 citeturn3view0

2. **参考图用几张最好？**  
   **答**：1–3 张；2–3 张可覆盖更多特征，通常一致性更好。 citeturn3view0

3. **可以用风格图做风格迁移吗？**  
   **答**：Veo 3.1 不支持 style reference；需用 Veo 2。 citeturn4view0

4. **输出只有 8 秒吗？如何做长视频？**  
   **答**：R2V 直接生成为 8 秒；用 Veo 3.1 的 Extend 每次 +7 秒，最多 20 次，合并上限约 148 秒。 citeturn4view0turn3view0

5. **支持竖屏和 1080p 吗？**  
   **答**：支持 16:9 与 9:16；分辨率可设到 1080p（新生）。 citeturn1view0turn9view0

6. **Extend 有什么限制？**  
   **答**：只能延长“Veo 自己生成”的视频；输入需 9:16 或 16:9、720p、≤141 秒。 citeturn3view0

7. **参考图尺寸/质量要求？**  
   **答**：JPEG/PNG；建议 720p+、主体清晰、背景简单，避免强遮挡。 citeturn4view0turn5search5

8. **大概多少钱？**  
   **答**：Google 开发者官方价：Veo 3 $0.40/秒，Veo 3 Fast $0.15/秒；第三方平台另计。 citeturn9view0

## 6. 竞品对比（简表）

- **Runway Gen-3（Act‑One/References）**：支持“角色参考 + 驱动表演”（上传驱动视频控制嘴型/表情）。更像动捕 + 参考图流程。适合对表演可控的镜头。 citeturn6search0turn6search6
- **Pika（PikaSwaps/参考换物）**：偏后期替换与局部编辑，可用参考图做对象替换；非严格的多图一致性工作流。 citeturn7search3turn7search1
- **Kling 1.6 Elements**：多参考图（最多 4）合成视频，更强的多素材拼接导向，但官方更偏 I2V/起始帧控制。 citeturn6search8

- 原生音频 + R2V + Extend 的整合工作流，适合“角色一致 + 长镜头拼接”的商业叙事。 citeturn1view0turn3view0
- 官方明确的 R2V 规范（1–3 张资产图）和 API 能力（9:16、1080p、降价）。 citeturn3view0turn9view0
