# Reference-to-Video 落地页需求规范

## 背景与目标

- 统一 `/reference-to-video` 落地页的内容结构与表现形式，方便后续在其他模型落地页复用。
- 明确各内容区域的产出要求，确保文案、媒体素材与交互行为保持一致性。
- 避免涉及组件或实现层面的技术方案，专注于内容与信息呈现层的需求描述。

## 页面区块总览

| 区块顺序 | 模块名称           | 必填/可选 | 内容来源/复用说明                                      |
| -------- | ------------------ | --------- | ------------------------------------------------------ |
| 1        | Hero               | 必填      | 专属文案与视频素材，复用视频特效页面的 VideoEffectHero |
| 2        | Feature Highlights | 必填      | 支持 1-n 个特性卡片，自定义配置                        |
| 3        | How to Use         | 必填      | 复用模型详情页 `How to Use` 模块内容结构               |
| 4        | FAQ + CTA          | 必填      | 复用 `/image-to-video` 页面现有内容                    |

## Hero 模块

- **组成**：主标题（H1）、副标题（H2/H3）、背景或内嵌视频、主 CTA 按钮（默认 `Try Reference-to-Video Now`）。
- **文案要求**：
  - 主标题突出“Reference-to-Video”与“Consistent Character”核心价值。
  - 副标题说明输入 1-3 张参考图后获得角色一致性视频的优势，建议 20-30 字。
- **媒体规范**：
  - 视频素材时长 ≤ 10s，循环播放，默认静音。
  - 若暂缺视频，可临时使用海报图（同一位置需预留 video 占位）。
- **交互期望**：
  - CTA 指向生成器锚点或页面内部生成入口。
  - Hero 下方默认展示关键信任标识（如模型版本标签、Beta 状态）可选。

## Feature Highlights 模块

- **结构**：支持配置 ≥1 个 Feature，每个 Feature 包含：
  - Feature 主标题：聚焦差异化能力，例如“角色一致性控制”。
  - Feature 副标题：补充场景或收益描述，建议 1-2 句。
  - 媒体：图片或视频（二选一、可为空）。
- **展示形式**：
  - 默认左右分栏布局，奇数序号媒体在右，偶数在左，以提升阅读节奏。
  - 支持通过配置选择是否显示媒体；无媒体时文案区域自适应全宽。
- **内容指南**：
  - 建议覆盖角色锁定、多素材输入、场景灵活度、质量等主题。
  - 每个 Feature 需附关键数据或用户收益要点（如“支持 1080p 输出”）。

## How to Use 模块

- **复用来源**：沿用模型详情页的 `How to Use` 模块结构与文案模板。
- **必要元素**：
  - 至少 3 个步骤（Step 1/2/3），描述上传参考图、撰写提示词、生成视频的流程。
  - 如需增加步骤，须确保语义完整且与实际操作一致。
- **视觉一致性**：
  - 使用与详情页相同的图标/编号样式，保持跨页面体验连贯。

## FAQ + CTA 模块

- **复用标准**：直接继承 `/image-to-video` 页面现有 FAQ 列表与 CTA 区块的内容结构。
- **可调参数**：
  - FAQ 若需新增问题，遵循原格式（问答结构，最多 6 条）。
  - CTA 文案可替换为与 Reference-to-Video 更贴合的行动语句，例如“Start Consistent Character Video Generation”。
- **定位**：FAQ 与 CTA 组合放置在页面底部，CTA 按钮指向生成器或注册流程。

## 内容交付清单

- Hero：主标题、副标题、视频源文件（或临时封面）、CTA 链接。
- Feature：至少 3 组 Feature 配置（标题、副标题、媒体资源、对应文案亮点）。
- How to Use：确认沿用的步骤文本与图标资源。
- FAQ + CTA：FAQ 文案校对完成，CTA 目标链接确认。

## 验收检查列表

- 页面加载后可依次看到 Hero、Feature、How to Use、FAQ、CTA 模块，排序无误。
- Hero 内视频自动播放、静音、循环，且在移动端 fallback 到封面图。
- Feature 区至少展示 1 条，媒体缺省时布局无断裂。
- How to Use 文案与模型详情页保持一致，步骤逻辑清晰。
- FAQ / CTA 与 `/image-to-video` 内容一致或经产品确认的差异项已标注。
