# veo3 双图上传与首尾帧支持规格

## 背景与目标

- 背景：现有 veo3 已支持上传两张照片并具备首尾帧能力，但前端图片上传组件与后端接口尚未统一支持双图工作流。
- 目标：当用户选择 veo3 视频模型时，上传组件允许上传两张图片，前端请求与后端接口以数组形式支持两张图片，并明确首帧/尾帧语义及处理路径；保持对单图与非 veo3 模型的兼容。

## 术语

- 首帧：视频合成的起始参考帧。
- 尾帧：视频合成的结束参考帧。
- 双图模式：同时上传首帧与尾帧两张参考图生成视频。

## 范围

- 在选择 veo3 视频模型时启用双图上传与首尾帧处理。
- 请求载荷、后端 API、服务层与模型调用流程的双图兼容。
- 其他模型与单图流程保持无破坏性向下兼容。

## 用户故事

- 作为用户，我在选择 veo3 后可以一次上传两张图片，系统将第一张作为首帧、第二张作为尾帧。
- 我可以只上传一张首帧图片，系统仅以首帧生成视频。
- 我可以预览、删除、重新排序两张图片（含一键交换顺序）。
- 提交生成后，能清楚看到处理状态与错误提示。

## 功能需求：前端 UI/交互

- 模型联动：当模型选择为 `veo3` 时，图片上传组件切换为“双图模式”；非 `veo3` 时维持单图模式。
- 上传限制：
  - 文件数：1–2 张（veo3）；其他模型：1 张。
  - 支持格式：jpeg/png/webp；最大体积与像素限制依现有限制并在 UI 标注。
- 预览与顺序：
  - 预览卡片显示“首帧”“尾帧”标记，支持拖拽排序或“交换顺序”按钮。
  - 删除任意一张后，剩余图片自动重排并更新标记。
- 校验与文案：
  - 超出数量、格式、大小即时报错，不进入提交。
  - 仅一张图时的提示文案说明“仅首帧将被使用”。
- 兼容与切换：
  - 从 veo3 切换为其他模型时，如已有两张图，保留第一张并提示“非 veo3 仅支持单图”；或根据现有产品策略清空第二张。
- 无障碍与键盘操作：预览卡片支持键盘焦点与操作提示。
