# Veo3 AI 业务功能详解

## 一、视频生成业务

### 1.1 核心业务流程

#### 视频生成工作流
```
用户请求 → 积分检查 → 提示词优化 → 模型选择 → 生成请求 
→ 状态追踪 → Webhook回调 → 结果存储 → 用户通知
```

### 1.2 支持的视频生成模式

#### 文本生成视频 (Text-to-Video)
- **输入**: 文本提示词
- **参数配置**:
  - 时长: 5秒、10秒
  - 宽高比: 16:9、9:16、1:1
  - 质量设置: cfg_scale
  - 负面提示词
- **输出**: MP4格式视频

#### 图片生成视频 (Image-to-Video)
- **输入**: 静态图片 + 动作提示词
- **支持格式**: JPG、PNG、WebP
- **特性**: 保持图片主体，添加动态效果
- **应用场景**: 产品展示、人物动画

### 1.3 多模型提供商系统

#### 当前激活的提供商

**1. Volcano Engine (Seedance模型)**
- 特点: 高质量视频生成，中文提示词支持好
- API密钥: ARK_API_KEY
- 请求限制: 并发限制
- 状态: ✅ 激活使用

**2. Kie.ai (Veo3模型)**
- 特点: Google Veo技术，细节表现力强
- API密钥: KIE_AI_API_KEY
- 特性: 支持高清升级
- 状态: ✅ 激活使用

**3. APICore (临时停用)**
- 特点: 多模型聚合
- 状态: ⏸️ 暂时注释

**4. fal.ai (临时停用)**
- 特点: 快速生成
- 状态: ⏸️ 暂时注释

### 1.4 提示词优化系统

#### 优化流程
1. **语言检测**: 识别输入语言
2. **关键词提取**: 提取核心概念
3. **风格增强**: 添加视觉风格描述
4. **技术参数**: 补充摄影技术词汇
5. **质量提升**: 添加质量修饰词

#### 优化示例
```
原始: "猫在玩球"
优化后: "A playful cat energetically batting a colorful ball, 
dynamic motion, cinematic lighting, 4K quality, smooth animation"
```

### 1.5 视频生成状态管理

#### 状态流转
- `PENDING`: 待处理
- `PROMPT_OPTIMIZING`: 提示词优化中
- `IN_QUEUE`: 排队中
- `IN_PROGRESS`: 生成中
- `COMPLETED`: 完成
- `FAILED`: 失败
- `SAVED_TO_R2`: 已存储到云端

#### 状态服务 (videoStatusService)
- 轮询检查生成状态
- Webhook实时更新
- 失败重试机制
- 超时处理

## 二、支付与订阅系统

### 2.1 积分系统

#### 积分定价
- **兑换率**: 0.25 USD = 10 积分
- **消耗规则**:
  - 标准视频生成: 10积分/次
  - 高清视频生成: 20积分/次
  - 提示词优化: 免费

#### 积分获取方式
1. **直接购买**: 一次性积分包
2. **订阅赠送**: 每月自动发放
3. **邀请奖励**: 成功邀请获得积分
4. **活动赠送**: 特殊活动奖励

### 2.2 订阅计划

#### 计划类型
```typescript
订阅等级:
- Starter: $9.99/月, 500积分
- Pro: $29.99/月, 2000积分
- Business: $99.99/月, 8000积分
- Enterprise: 定制方案
```

#### 订阅特权
- 每月自动充值积分
- 优先生成队列
- 高清视频导出
- 批量生成支持
- API访问权限

### 2.3 支付路由系统

#### PaymentRouter架构
```typescript
地理位置检测 → 支付方式推荐 → 用户选择 → 支付处理
```

#### 支付提供商选择逻辑
1. **俄罗斯地区**: Payssion (SberPay、YooMoney、Mir Card)
2. **国际地区**: 
   - 优先: Creem (最新集成)
   - 备选: Stripe
3. **中国地区**: 特殊处理流程

### 2.4 支付安全

#### Webhook验证
- **Payssion**: HMAC-SHA256签名验证
- **Stripe**: Webhook密钥验证
- **Creem**: API密钥验证

#### 订单处理流程
1. 创建待支付订单
2. 跳转支付页面
3. 支付回调验证
4. 订单状态更新
5. 积分/会员发放
6. 发送确认邮件
