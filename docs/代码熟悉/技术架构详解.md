# Veo3 AI 技术架构详解

## 一、系统架构概览

### 1.1 整体架构图

```
┌──────────────────────────────────────────────────────┐
│                     前端应用层                         │
│  Next.js 14 App Router + React 18 + TypeScript       │
├──────────────────────────────────────────────────────┤
│                     API网关层                         │
│         Next.js API Routes + Middleware              │
├──────────────────────────────────────────────────────┤
│                     业务服务层                        │
│   Payment Service | Video Service | User Service     │
├──────────────────────────────────────────────────────┤
│                     数据访问层                        │
│      Supabase Client | External APIs | Storage       │
├──────────────────────────────────────────────────────┤
│                    基础设施层                         │
│   PostgreSQL | Cloudflare R2 | Redis (规划中)        │
└──────────────────────────────────────────────────────┘
```

### 1.2 技术选型理由

| 技术选择 | 选择理由 |
|---------|---------|
| Next.js 14 | App Router提供更好的性能，RSC减少客户端负载 |
| TypeScript | 类型安全，提高代码质量和开发效率 |
| Supabase | 开箱即用的BaaS，内置RLS安全机制 |
| Cloudflare Pages | 全球CDN，边缘计算，成本效益高 |
| Tailwind CSS | 原子化CSS，快速开发，性能优异 |
| NextAuth.js | 灵活的认证方案，支持多种提供商 |

#### 路由组设计理念
- **路由组隔离**: 使用括号语法创建逻辑分组
- **布局复用**: 每个路由组可有独立布局
- **权限控制**: 在layout中统一处理认证

export async function requireAuth(request: Request) {
  const session = await auth();
  
  if (!session) {
    throw new UnauthorizedError();
  }
  
  return session;
}

export async function requireAdmin(request: Request) {
  const session = await requireAuth(request);
  
  if (session.user.role !== 'admin') {
    throw new ForbiddenError();
  }
  
  return session;
}

// app/api/payssion/v2-webhook/route.ts
export async function POST(request: Request) {
  // 1. 验证签名
  const signature = request.headers.get('x-signature');
  if (!verifySignature(body, signature)) {
    return unauthorized();
  }
  
  // 2. 处理支付事件
  switch (body.event) {
    case 'payment.completed':
      await handlePaymentCompleted(body);
      break;
    case 'payment.failed':
      await handlePaymentFailed(body);
      break;
  }
  
  return NextResponse.json({ received: true });
}

export async function saveVideoToR2(
  videoUrl: string,
  userId: string
): Promise<string> {
  const response = await fetch(videoUrl);
  const blob = await response.blob();
  
  const key = `videos/${userId}/${Date.now()}.mp4`;
  
  await r2Client.put(key, blob, {

    },
  });
  
  return `${R2_PUBLIC_URL}/${key}`;

### 9.2 性能监控

```typescript
// 性能追踪
export async function trackPerformance(
  operation: string,
  fn: () => Promise<any>
) {
  const start = performance.now();
  
  try {
    const result = await fn();
    const duration = performance.now() - start;
    
    analytics.track('performance', {
      operation,
      duration,
      success: true
    });
    
    return result;
  } catch (error) {
    const duration = performance.now() - start;
    
    analytics.track('performance', {
      operation,
      duration,
      success: false,
      error: error.message
    });
    
    throw error;
  }
}
```
