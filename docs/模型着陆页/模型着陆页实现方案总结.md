# 模型着陆页实现方案总结

## 总体结构

- **统一动态路由**：使用 `app/[locale]/(default)/[model]/page.tsx` 处理所有模型着陆页
- **配置驱动**：模型列表定义在 `config/model-landing-pages.ts` 中
- **数据加载**：`services/page.ts` 的 `getModelLandingPage(model, locale)` 统一处理数据加载
- **类型安全**：所有数据解析为 `ModelLandingPage` 类型
- **自动渲染**：展示层由 `components/blocks/model-landing-page` 组件组合，缺少数据的区块自动跳过

## 核心文件

### 配置文件：`config/model-landing-pages.ts`

定义所有支持的模型 slug：

```typescript
export const MODEL_LANDING_PAGES = [
  'nano-banana',
  'wan-2-5',
] as const;
```

### 数据服务：`services/page.ts`

统一的数据加载函数：

```typescript
export async function getModelLandingPage(
  model: string,
  locale: string
): Promise<ModelLandingPage>
```

- 自动处理 `zh-CN` → `zh` 规范化
- 失败时自动回退到英文（`en.json`）

### 动态路由：`app/[locale]/(default)/[model]/page.tsx`

- **路由生成**：`generateStaticParams()` 基于配置自动生成所有 model × locale 组合
- **SEO 元数据**：`generateMetadata()` 从 `i18n/messages/<locale>.json` 读取 SEO 信息
- **数据验证**：通过 `isValidModelSlug()` 验证，无效 slug 返回 404

## 已有模型页面

当前支持的模型：

- **Nano Banana**：`nano-banana`
- **Wan 2.5**：`wan-2-5`

## 共享 UI 组件

位于 `components/blocks/model-landing-page/`：

- `model-banner.tsx`：Tab 切换文生图/图生图，处理 Prompt 或图片上传
- `model-key-features.tsx`：渲染特性简表与详细内容，支持表格、图片、视频
- `model-usage-guide.tsx`：使用步骤说明
- `model-cta.tsx`：回到顶部按钮
- `partners-scroll.tsx`：合作伙伴展示
- `youtube-case-show.tsx`、`reddit-case-show.tsx`、`twitter-case-show.tsx`：案例展示
- `index.tsx`：根据数据存在性串联上述所有区块

## 数据约定

### JSON 数据结构

- **类型定义**：`types/pages/model-landing-page.ts`
- **包含字段**：Banner tabs、合作伙伴、特性列表、步骤说明、案例列表、FAQ、CTA 等
- **可选字段**：JSON 中缺失的字段对应区块不渲染
- **必要字段**：保持基础字段完整（如 `banner.title`、`usageGuide.steps`）避免组件报错

### i18n 命名规范

**重要**：model slug 与 i18n key 必须完全一致

```json
// i18n/messages/en.json
{
  "nano-banana": {  // ✅ 与 slug 一致
    "title": "...",
    "description": "...",
    "keywords": "..."
  },
  "wan-2-5": {      // ✅ 与 slug 一致
    "title": "...",
    "description": "...",
    "keywords": "..."
  }
}
```

## 新增模型页面流程

### 只需 3 步！

#### 1. 添加模型配置

在 `config/model-landing-pages.ts` 中添加 slug：

```typescript
export const MODEL_LANDING_PAGES = [
  'nano-banana',
  'wan-2-5',
  'new-model',  // 👈 添加新模型
] as const;
```

#### 2. 准备 JSON 数据文件

创建数据文件：
- `i18n/pages/new-model/en.json`
- `i18n/pages/new-model/ru.json`（可选，支持多语言）

参考现有模板（`nano-banana/en.json`），按 `ModelLandingPage` 类型填充内容。

#### 3. 添加 SEO 元数据

在 `i18n/messages/en.json` 和 `i18n/messages/ru.json` 中添加：

```json
{
  "new-model": {  // 👈 key 必须与 slug 完全一致
    "title": "New Model - AI Image Generator | Veo3 AI",
    "description": "...",
    "keywords": "..."
  }
}
```

#### 4. 验证

```bash
pnpm dev
# 访问 http://localhost:3000/new-model
# 访问 http://localhost:3000/ru/new-model
```

### 完成！

无需创建任何路由文件或数据加载函数，系统自动处理：
- ✅ 路由生成（`/new-model`, `/ru/new-model`）
- ✅ SEO metadata
- ✅ 数据加载和渲染

## 架构优势

1. **零重复代码**：所有模型共享同一套路由逻辑
2. **配置驱动**：新增模型只需修改配置，不需要写代码
3. **类型安全**：TypeScript 确保配置完整性
4. **易于维护**：单一数据源，修改影响所有模型
5. **命名统一**：消除了 slug 到 i18n key 的映射层
