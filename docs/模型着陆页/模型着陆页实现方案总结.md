# 模型着陆页实现方案总结

## 总体结构

- 路由入口位于 `app/[locale]/(default)/<model>/page.tsx`，负责生成 metadata 并加载模型页数据。
- 数据通过 `services/page.ts` 的动态 `import` 从 `i18n/pages/<model>/<locale>.json` 读取，统一解析为 `ModelLandingPage` 类型。
- 展示层由 `components/blocks/model-landing-page` 下的组件组合完成，缺少对应数据的区块会被自动跳过。

## 已有模型页面

- **Nano Banana**：`app/[locale]/(default)/nano-banana/page.tsx` → `getNanoBananaLandingPage(locale)` → `i18n/pages/nano-banana/<locale>.json`。
- **Wan 2.5**：`app/[locale]/(default)/wan-2-5/page.tsx` → `getWanAILandingPage(locale)` → `i18n/pages/wan-2-5/<locale>.json`。

两者共享的 UI 组件：

- `model-banner.tsx`：Tab 切换文生图/图生图，处理 Prompt 或图片上传，保存到 `localStorage` 后跳转生成页面。
- `model-key-features.tsx`：渲染特性简表与详细内容，支持表格、图片、视频展示。
- `model-usage-guide.tsx`、`model-cta.tsx`：提供使用步骤和回到顶部按钮。
- `partners-scroll.tsx`、`youtube-case-show.tsx`、`reddit-case-show.tsx`、`twitter-case-show.tsx`：展示合作伙伴、视频案例以及 Reddit/Twitter iframe。
- `components/blocks/model-landing-page/index.tsx`：根据数据存在性串联上述所有区块。

## 数据约定

- `types/pages/model-landing-page.ts` 定义了所有字段：Banner tabs、合作伙伴、特性列表、步骤说明、案例列表、FAQ、CTA 等。
- JSON 中缺失的字段会导致对应区块不渲染，但应保持基础字段（如 `banner.title`、`usageGuide.steps`）完整，以避免组件报错。
- `services/page.ts` 会将 `zh-CN` 正规化为 `zh`，并在缺失指定语言文件时回退英文。

## 新增模型页面流程

1. **准备文案**：在 `i18n/pages/<new-model>/en.json` 复制现有模板，按 `ModelLandingPage` 结构填充内容；如需多语言，继续添加 `<locale>.json`。
2. **新增数据访问函数**：在 `services/page.ts` 中仿照 `getNanoBananaLandingPage` 新增 `get<NewModel>LandingPage`。
3. **更新页面翻译（TDK）**：在 `i18n/[locale].json` 或相应的 messages 文件中补充 `new_model.title` / `new_model.description` / `new_model.keywords`，供 `generateMetadata` 使用。
4. **创建路由**：复制 `nano-banana/page.tsx`，替换导入的 getter、翻译 key 以及 canonical 链接路径。
5. **校验**：运行 `pnpm dev` 检查页面渲染是否符合预期；必要时补充翻译文件中的 metadata 字段或图片资源。
