# Toast显示图片地址功能

## 🎯 功能概述

优化图片生成成功后的用户反馈，在Toast提示中直接显示生成的图片地址，让用户能够立即看到并使用生成的图片URL。

## ✨ 功能特点

### 1. **直观的URL显示**
- 📋 **完整地址**: Toast中显示完整的图片URL
- 👁️ **即时可见**: 生成完成后立即显示
- ⏰ **延长显示**: 5秒显示时间，确保用户有足够时间查看

### 2. **增强的用户体验**
- 🔗 **地址可见**: 用户无需猜测图片位置
- 📱 **便于分享**: 可以直接从Toast中看到分享链接
- ✅ **操作确认**: 明确显示生成成功和URL获取

## 🔧 技术实现

### 核心代码
```typescript
// Auto copy image URL to clipboard
try {
  await navigator.clipboard.writeText(imageUrl);
  // Show toast with the actual image URL
  toast.success(`Image generated! URL: ${imageUrl}`, {
    duration: 5000, // Show for 5 seconds
  });
} catch (error) {
  console.error("Failed to copy URL:", error);
  // Show toast with the actual image URL even if copy failed
  toast.success(`Image generated! URL: ${imageUrl}`, {
    duration: 5000,
  });
}
```

### 关键特性
- ✅ **URL显示**: 无论复制是否成功，都显示图片URL
- ⏰ **延长时间**: 5秒显示时间，足够用户查看完整URL
- 🛡️ **错误处理**: 即使复制失败也会显示URL

## 💡 设计考虑

### 1. **信息层次**
- 🎯 **主要信息**: "Image generated!" - 告知操作成功
- 🔗 **次要信息**: "URL: https://..." - 提供具体地址
- ⏰ **显示时长**: 5秒 - 平衡可读性和界面干净度

### 2. **错误处理**
```typescript
try {
  // 尝试复制到剪贴板
  await navigator.clipboard.writeText(imageUrl);
  // 成功: 显示URL
  toast.success(`Image generated! URL: ${imageUrl}`, { duration: 5000 });
} catch (error) {
  // 失败: 仍然显示URL，但不提及复制成功
  toast.success(`Image generated! URL: ${imageUrl}`, { duration: 5000 });
}
```

### 3. **一致性保证**
- 📱 **统一格式**: 所有场景都使用相同的Toast格式
- ⏰ **统一时长**: 所有URL展示都使用5秒时长
- 🎨 **统一样式**: 保持现有的Toast样式规范
