## 数据库架构概述

VEO3 项目使用 **Supabase** 作为主要数据库服务，提供：
- **PostgreSQL** 数据库
- **认证服务** (Supabase Auth)
- **实时数据** (Realtime)
- **存储服务** (Storage)
- **边缘函数** (Edge Functions)

### 主要特性
- 🔐 **多重认证**: 支持邮箱、Google、GitHub、Apple 登录
- 💳 **支付集成**: Stripe、Creem、Payssion V2
- 🎬 **视频生成**: 多提供商支持 (Volcano、Kie.ai、fal.ai、Ali)
- 🖼️ **图片生成**: Nano Banana API 集成
- 👥 **用户管理**: 积分系统、会员制、邀请码

## Supabase 配置

### 1. 注册 Supabase 账户
1. 访问 [Supabase 官网](https://supabase.com)
2. 创建新项目
3. 选择地区（建议选择离用户最近的地区）
4. 等待项目初始化完成

### 2. 获取连接信息
在 Supabase Dashboard 中：
- **Project URL**: 在 Settings > API 中找到
- **API Keys**: 
  - `anon` key: 客户端使用
  - `service_role` key: 服务端使用（敏感操作）

### 3. 数据库连接字符串
```
postgresql://postgres:[YOUR-PASSWORD]@db.[PROJECT-REF].supabase.co:5432/postgres
```

## 环境变量设置

### 创建环境变量文件
在项目根目录创建 `.env.local` 文件：

```bash
# ===========================================
# 核心数据库配置
# ===========================================
SUPABASE_URL=https://your-project-ref.supabase.co
SUPABASE_ANON_KEY=your_anon_key_here
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key_here

# ===========================================
# 认证配置
# ===========================================
NEXTAUTH_SECRET=your_nextauth_secret_here
NEXTAUTH_URL=http://localhost:3000

# Google OAuth
GOOGLE_CLIENT_ID=your_google_client_id
GOOGLE_CLIENT_SECRET=your_google_client_secret

# GitHub OAuth
GITHUB_CLIENT_ID=your_github_client_id
GITHUB_CLIENT_SECRET=your_github_client_secret

# Apple OAuth
APPLE_CLIENT_ID=your_apple_client_id
APPLE_CLIENT_SECRET=your_apple_client_secret

# ===========================================
# 支付服务配置
# ===========================================
# Stripe
STRIPE_PUBLIC_KEY=pk_test_...
STRIPE_PRIVATE_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...

# Creem
CREEM_API_KEY=your_creem_api_key
CREEM_WEBHOOK_SECRET=your_creem_webhook_secret

# Payssion V2
PAYSSION_API_KEY=your_payssion_api_key
PAYSSION_SECRET_KEY=your_payssion_secret_key

# ===========================================
# 视频生成服务
# ===========================================
# Volcano Engine (Seedance)
ARK_API_KEY=your_volcano_api_key

# Kie.ai
KIE_AI_API_KEY=your_kie_ai_api_key

# fal.ai
FAL_KEY=your_fal_api_key

# Ali Cloud
ALI_API_KEY=your_ali_api_key

# ===========================================
# 存储配置
# ===========================================
STORAGE_ENDPOINT=your_r2_endpoint
STORAGE_REGION=auto
STORAGE_ACCESS_KEY=your_access_key
STORAGE_SECRET_KEY=your_secret_key
STORAGE_BUCKET=your_bucket_name
STORAGE_DOMAIN=your_custom_domain

# ===========================================
# 分析服务
# ===========================================
NEXT_PUBLIC_YANDEX_METRICA_ID=your_yandex_id
YANDEX_OFFLINE_CONVERSION_TOKEN=your_yandex_token
NEXT_PUBLIC_PLAUSIBLE_DOMAIN=your_domain

# ===========================================
# 应用配置
# ===========================================
NEXT_PUBLIC_WEB_URL=http://localhost:3000
NODE_ENV=development
```

### 环境变量说明

#### 必需变量
- `SUPABASE_URL`: Supabase 项目 URL
- `SUPABASE_ANON_KEY`: 客户端 API 密钥
- `NEXTAUTH_SECRET`: 认证加密密钥
- `NEXTAUTH_URL`: 应用 URL

#### 可选变量
- OAuth 提供商密钥（根据需要配置）
- 支付服务密钥（测试环境可用测试密钥）
- 视频生成服务密钥（至少配置一个）

## 数据库表结构

### 核心表

#### 1. users 表
```sql
CREATE TABLE users (
    uuid VARCHAR(255) PRIMARY KEY,
    email VARCHAR(255) UNIQUE NOT NULL,
    name VARCHAR(255),
    image VARCHAR(500),
    provider VARCHAR(50),
    email_verified BOOLEAN DEFAULT false,
    is_admin BOOLEAN DEFAULT false,
    inviter_uuid VARCHAR(255),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 2. video_generations 表
```sql
CREATE TABLE video_generations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id TEXT NOT NULL REFERENCES users(uuid),
    model_id TEXT NOT NULL,
    prompt TEXT NOT NULL,
    optimized_prompt TEXT,
    fal_request_id TEXT,
    volcano_request_id TEXT,
    veo3_request_id TEXT,
    ali_request_id TEXT,
    input_image_url TEXT,
    negative_prompt TEXT,
    aspect_ratio TEXT DEFAULT '16:9',
    duration_seconds INTEGER DEFAULT 5,
    cfg_scale DECIMAL,
    seed INTEGER,
    has_audio BOOLEAN DEFAULT false,
    status video_generation_status DEFAULT 'PENDING',
    video_url_r2 TEXT,
    video_url_fal TEXT,
    error_message TEXT,
    logs JSONB,
    metrics JSONB,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 状态枚举
CREATE TYPE video_generation_status AS ENUM (
    'PENDING',
    'IN_QUEUE',
    'IN_PROGRESS',
    'COMPLETED',
    'FAILED'
);
```

#### 3. orders 表
```sql
CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_uuid VARCHAR(255) REFERENCES users(uuid),
    stripe_session_id VARCHAR(255),
    payssion_transaction_id VARCHAR(255),
    creem_session_id VARCHAR(255),
    product_id VARCHAR(255) NOT NULL,
    amount DECIMAL(10,2) NOT NULL,
    currency VARCHAR(3) DEFAULT 'USD',
    status VARCHAR(50) DEFAULT 'pending',
    payment_provider VARCHAR(50) DEFAULT 'stripe',
    payssion_mandate_id VARCHAR(255),
    subscription_status VARCHAR(50),
    next_billing_date TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 4. memberships 表
```sql
CREATE TABLE memberships (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_uuid VARCHAR(255) REFERENCES users(uuid),
    plan_id VARCHAR(255) NOT NULL,
    status VARCHAR(50) DEFAULT 'active',
    current_period_start TIMESTAMPTZ,
    current_period_end TIMESTAMPTZ,
    cancel_at_period_end BOOLEAN DEFAULT false,
    stripe_subscription_id VARCHAR(255),
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 5. credits 表
```sql
CREATE TABLE credits (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_uuid VARCHAR(255) REFERENCES users(uuid),
    amount INTEGER NOT NULL,
    type VARCHAR(50) NOT NULL, -- 'purchase', 'gift', 'refund', 'usage'
    description TEXT,
    order_id UUID REFERENCES orders(id),
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 支付相关表

#### 6. payssion_mandates 表 (俄罗斯支付)
```sql
CREATE TABLE payssion_mandates (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_uuid VARCHAR(255) REFERENCES users(uuid),
    user_email VARCHAR(255) NOT NULL,
    mandate_id VARCHAR(255) UNIQUE NOT NULL,
    payment_method VARCHAR(50) NOT NULL, -- mir, yoomoney, sberpay, tbank
    status VARCHAR(50) DEFAULT 'pending', -- pending, authorized, expired, canceled
    authorization_url TEXT,
    authorized_at TIMESTAMPTZ,
    expires_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

#### 7. creem_subscriptions 表
```sql
CREATE TABLE creem_subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_uuid VARCHAR(255) REFERENCES users(uuid),
    subscription_id VARCHAR(255) UNIQUE NOT NULL,
    plan_id VARCHAR(255) NOT NULL,
    status VARCHAR(50) DEFAULT 'active',
    current_period_start TIMESTAMPTZ,
    current_period_end TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 索引优化
```sql
-- 用户相关索引
CREATE INDEX idx_users_email ON users(email);
CREATE INDEX idx_users_provider ON users(provider);

-- 视频生成索引
CREATE INDEX idx_video_generations_user_id ON video_generations(user_id);
CREATE INDEX idx_video_generations_status ON video_generations(status);
CREATE INDEX idx_video_generations_created_at ON video_generations(created_at DESC);

-- 订单索引
CREATE INDEX idx_orders_user_uuid ON orders(user_uuid);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_orders_payment_provider ON orders(payment_provider);

-- 积分索引
CREATE INDEX idx_credits_user_uuid ON credits(user_uuid);
CREATE INDEX idx_credits_type ON credits(type);
CREATE INDEX idx_credits_created_at ON credits(created_at DESC);
```

## 本地开发设置

### 1. 安装依赖
```bash
# 使用 pnpm 安装依赖
pnpm install
```

### 2. 数据库初始化

#### 方法一: 使用 Supabase CLI (推荐)
```bash
# 安装 Supabase CLI
npm install -g supabase

# 登录 Supabase
supabase login

# 链接到现有项目
supabase link --project-ref your-project-ref

# 生成类型定义
supabase gen types typescript --project-id your-project-ref > types/supabase.ts
```

#### 方法二: 手动执行 SQL
1. 在 Supabase Dashboard 中打开 SQL Editor
2. 复制并执行上面的表结构 SQL
3. 创建必要的索引和约束

### 3. 启动开发服务器
```bash
# 启动开发服务器
pnpm dev

# 或指定端口
pnpm dev --port 3001
```

### 4. 验证连接
访问 `http://localhost:3000` 并尝试：
- 用户注册/登录
- 查看控制台是否有数据库连接错误

## 常见问题解决

### 1. 连接失败
**错误**: `Supabase URL or key is not set`
**解决**: 
- 检查 `.env.local` 文件是否存在
- 确认环境变量名称正确
- 重启开发服务器

### 2. 认证问题
**错误**: NextAuth 相关错误
**解决**:
- 确保 `NEXTAUTH_SECRET` 已设置
- 检查 OAuth 提供商配置
- 确认回调 URL 正确

### 3. 数据库权限问题
**错误**: `permission denied for table`
**解决**:
- 使用 `service_role` key 进行管理操作
- 检查 RLS (Row Level Security) 策略
- 确认用户权限设置

### 4. 端口占用
**错误**: `EADDRINUSE: address already in use`
**解决**:
```bash
# 查找占用端口的进程
lsof -i :3000

# 结束进程
kill -9 <PID>

# 或使用不同端口
pnpm dev --port 3001
```

### 5. 环境变量不生效
**解决**:
- 确认文件名为 `.env.local`
- 重启开发服务器
- 检查变量名称（不要有额外空格）

### 生产环境配置
1. **使用生产数据库**: 不要在生产环境使用开发数据库
2. **环境变量安全**: 使用安全的密钥管理服务
3. **数据备份**: 定期备份生产数据
4. **监控设置**: 配置数据库性能监控

### 性能优化
1. **连接池**: 配置合适的连接池大小
2. **查询优化**: 使用 EXPLAIN 分析慢查询
3. **索引优化**: 根据查询模式优化索引
4. **缓存策略**: 合理使用 Redis 等缓存
