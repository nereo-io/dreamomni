## 📊 创建的表结构

### image_generations 表
- **主键**: `id` (UUID)
- **外键**: `user_id` → `auth.users(id)`
- **核心字段**:
  - `prompt` (TEXT) - 用户输入的提示词
  - `status` (ENUM) - 生成状态
  - `image_urls` (TEXT[]) - 生成的图片URL数组
  - `model_id` (VARCHAR) - 使用的AI模型
  - `mode` (ENUM) - 生成模式 (text-to-image, image-edit)
  - `credits_used` (INTEGER) - 消耗的积分

### image_generation_credits 表
- **主键**: `id` (UUID)
- **外键**: 
  - `image_generation_id` → `image_generations(id)`
  - `user_id` → `auth.users(id)`
- **核心字段**:
  - `credits_consumed` (INTEGER) - 消耗的积分数量
  - `credits_cost` (DECIMAL) - 积分成本

## 🔧 故障排除

### 如果表创建失败

1. **检查权限**
   - 确认您在Supabase项目中有管理员权限
   - 确认 `SUPABASE_SERVICE_ROLE_KEY` 正确

2. **分步创建**
   如果整个SQL脚本执行失败，可以分步执行：

   ```sql
   -- 第一步：创建枚举类型
   CREATE TYPE image_generation_status AS ENUM (
       'PENDING', 'IN_QUEUE', 'IN_PROGRESS', 'COMPLETED', 'FAILED', 'SAVED_TO_R2'
   );
   
   -- 第二步：创建主表
   CREATE TABLE image_generations (
       id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
       user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
       -- ... 其他字段
   );
   ```

3. **检查现有表**
   在Supabase Dashboard的Table Editor中查看是否有部分表已创建
