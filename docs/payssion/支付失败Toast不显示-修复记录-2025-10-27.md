# Payssion 支付失败 Toast 不显示 - 修复记录

**日期：** 2025-10-27
**问题：** 用户在 /pricing 页面点击支付（existing mandate），余额不足导致支付失败，但前端没有显示错误 Toast
**状态：** ✅ 已修复

---

## 问题回顾

### 症状
用户录屏显示：
- 点击支付按钮
- 有 existing mandate，不跳转到 Payssion 页面
- 直接扣款
- 扣款失败（余额不足）
- **没有显示任何错误提示** ❌

### 预期行为
应该显示俄语 Toast：
```
"Недостаточно средств на счёте. Пополните баланс и попробуйте снова."
```

### 3. 定位根本原因

既然后端记录正常，问题只能在**前端检查逻辑**。

检查 `app/api/check-recent-payment/route.ts`：

```typescript
// Line 50: 查询最近 5 分钟的失败订单
const recentFailure = await findRecentFailedPayment(user_uuid, 5);

// Line 75: 但只返回 1 分钟内的失败 ❌
if (failureTime >= requestTimestamp - 60000) {
  return Toast;
}
```

**发现问题：**
- 查询窗口：5 分钟 ✅
- 匹配窗口：1 分钟 ❌ **太窄了！**

**为什么会超过 1 分钟？**
1. Payssion 扣款延迟（10-30 秒）
2. Webhook 网络传输延迟
3. 用户浏览器时间可能不准
4. 用户可能等几秒才检查

---

## 修复方案

### 核心修复：扩大时间匹配窗口

**从 1 分钟扩大到 5 分钟，与查询窗口一致**

### 附加改进：添加详细调试日志

为了以后能快速定位问题，在关键位置添加日志。

## 后续优化建议

### 1. 考虑移除时间窗口检查

既然查询已经限制在 5 分钟内，时间匹配检查可能是多余的：

```typescript
// 简化版本：移除时间检查
const recentFailure = await findRecentFailedPayment(user_uuid, 5);

if (recentFailure) {
  // 直接返回，不检查时间窗口
  return respData({
    hasFailedPayment: true,
    failureInfo: {...}
  });
}
```

**优点：**
- 逻辑更简单
- 不依赖浏览器时间
- 避免时区问题

**缺点：**
- 如果用户在 5 分钟内多次支付，可能返回旧的失败记录

### 2. 添加邮件通知

对于自动续费失败（用户不在页面上），应该发送邮件：

```typescript
// PayssionProvider.ts handlePaymentFailed
if (subscriptionId) {
  // 发送邮件通知
  await sendPaymentFailureEmail(
    metadata.user_email,
    displayMessage,
    subscriptionId
  );
}
```

### 3. 在 Dashboard 显示横幅

对于 `past_due` 状态的订阅，在用户下次登录时显示警告：

```tsx
// dashboard/layout.tsx
{subscription?.status === 'past_due' && (
  <Alert variant="destructive">
    <AlertCircle />
    <AlertTitle>Payment Failed</AlertTitle>
    <AlertDescription>
      Your last payment failed. Please update your payment method.
    </AlertDescription>
  </Alert>
)}
```
