**一、整体方案概述**

1.  **数据库 (Supabase)**:

    - 原图和水印图都存储在 Supabase Storage 中。
    - 水印图可以公开访问。
    - 原图需要保护，只有付费用户才能通过生成的安全下载链接（Signed URL）进行下载。
    - 水印可以在上传原图后通过后端逻辑（如 Supabase Edge Function）自动生成，或者由管理员手动上传两个版本。
3.  **后端 API (Next.js API Routes)**:
    - `/api/products`: 获取商品列表。

    - `/api/webhooks/stripe`: (扩展现有 Webhook) 处理支付成功后的回调，记录购买，并授予用户下载权限。
    - `/api/products/[productId]/download`: 用户付费后，调用此接口获取原图的安全下载链接。
4.  **前端页面与组件 (Next.js & React)**:
    - **商品列表页**: `app/[locale]/(default)/products/page.tsx`，展示所有商品的卡片（带水印图、名称、价格）。
    - **商品详情页**: `app/[locale]/(default)/products/[productId]/page.tsx`，展示商品大水印图、详情描述、价格和购买按钮。

    - **相关组件**: `ProductCard.tsx`, `ProductDetailView.tsx` 等，放在 `components/products/` 目录下。
5.  **支付集成 (Stripe)**:
    - 复用或扩展现有的 Stripe 支付流程。
6.  **国际化 (next-intl)**:
    - 所有新的 UI 文本都需要添加到相应的翻译文件中。
